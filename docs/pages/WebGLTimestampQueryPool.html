WebGLTimestampQueryPool

管理用于性能测量的 WebGL 时间戳查询池。 使用 WebGL 扩展处理计时查询的创建、执行与解析。

构造函数

new WebGLTimestampQueryPool( gl : WebGLRenderingContext | WebGL2RenderingContext, type : string, maxQueries : number )

创建一个新的 WebGL 时间戳查询池。

gl

WebGL 上下文。
type

此查询池的类型标识符。
maxQueries

此池可容纳的最大查询数量。

默认值为 2048。

方法

.allocateQueriesForContext( uid : string ) : number

为给定的渲染上下文分配一对查询。

uid

渲染上下文的唯一标识符。
重写： TimestampQueryPool#allocateQueriesForContext
返回值： 已分配查询的基偏移量；若分配失败则返回 null。

.beginQuery( uid : string )

为指定的渲染上下文开始一次时间戳查询。

uid

渲染上下文的唯一标识符。

.dispose()

释放此查询池持有的所有资源。 包括删除所有查询对象并清除内部状态。

重写： TimestampQueryPool#dispose

.endQuery( uid : string )

结束指定渲染上下文的活动时间戳查询。

uid

渲染上下文的唯一标识符。

.resolveQueriesAsync() : Promise.<number> (async)

异步解析所有已完成的查询，并返回总时长。

重写： TimestampQueryPool#resolveQueriesAsync
返回值： 总时长（毫秒）；若解析失败则返回上一个有效值。

.resolveQuery( query : WebGLQuery ) : Promise.<number> (async)

解析单个查询，检查是否完成以及是否发生不相交操作。

query

要解析的查询对象。
返回值： 经过的时间（毫秒）。
