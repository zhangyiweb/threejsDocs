LightShadow

各类灯光阴影配置的抽象基类。这些类 表示不同光源类型的阴影配置。

构造函数

new LightShadow( camera : Camera ) (abstract)

构造一个新的灯光阴影。

camera

该灯光对世界的观察视角。

属性

.autoUpdate : boolean

启用灯光阴影的自动更新。若不需要动态 光照 / 阴影，可将此属性设为 false。

默认值为 true。

.bias : number

阴影贴图偏移量，在判断表面是否处于阴影中时， 对归一化深度进行加减的量。

默认值为 0。此处极小的调整（约 0.0001 量级） 可能有助于减少阴影伪影。

默认值为 0。

.biasNode : Node.<float>

bias 的节点版本。仅支持 WebGPURenderer。

若定义了 bias 节点，则 bias 无效。

默认值为 null。

.blurSamples : number

对 VSM 阴影贴图进行模糊时使用的采样数量。

默认值为 8。

.camera : Camera

该灯光对世界的观察视角。

.intensity : number

阴影强度。默认值为 1。 有效取值范围为 [0, 1]。

默认值为 1。

.map : RenderTarget

使用内部相机生成的深度贴图；像素深度之外的位置 处于阴影中。在渲染过程中由内部计算。

默认值为 null。

.mapPass : RenderTarget

使用内部相机生成的分布贴图；根据深度分布 计算遮挡。在渲染过程中由内部计算。

默认值为 null。

.mapSize : Vector2

定义阴影贴图的宽度和高度。值越高阴影质量越好， 但计算耗时也越长。值必须为 2 的幂。

默认值为 (512,512)。

.mapType : number

阴影纹理类型。默认值为 UnsignedByteType。

默认值为 UnsignedByteType。

.matrix : Matrix4

从模型空间到阴影相机空间的变换矩阵，用于计算阴影贴图中的位置与深度。 在渲染过程中由内部计算。

.needsUpdate : boolean

设为 true 时，阴影贴图将在下一次 render 调用中更新。 若已将 LightShadow#autoUpdate 设为 false，则需要 将此属性设为 true，再执行一次渲染调用以更新灯光阴影。

默认值为 false。

.normalBias : number

定义查询阴影贴图时沿物体法线偏移位置的量。 默认值为 0。增大该值可用于减少阴影痤疮，尤其在大型场景中 光线以浅角度照射几何体时。代价是阴影可能出现扭曲。

默认值为 0。

.radius : number

设为大于 1 的值会使阴影边缘模糊。 过高的值会导致阴影出现不必要的条带效应——在这些效应 变得可见之前，更大的贴图尺寸允许使用更高的值。

当阴影贴图类型为 BasicShadowMap 时，此属性无效。

默认值为 1。

方法

.clone() : LightShadow

返回一个新的灯光阴影实例，其值从本实例复制而来。

返回值： 本实例的克隆。

.copy( source : LightShadow ) : LightShadow

将给定灯光阴影实例的值复制到本实例。

source

要复制的灯光阴影。
返回值： 对本灯光阴影实例的引用。

.dispose()

释放本实例分配的 GPU 相关资源。当应用中 不再使用本实例时，请调用此方法。

.getFrameExtents() : Vector2

返回帧扩展范围。

返回值： 帧扩展范围。

.getFrustum() : Frustum

获取阴影相机的视锥体。由渲染器内部用于剔除物体。

返回值： 阴影相机的视锥体。

.getViewport( viewportIndex : number ) : Vector4

返回给定视口索引的视口定义。

viewportIndex

视口索引。
返回值： 视口。

.getViewportCount() : number

由渲染器内部使用，以获取此阴影需要 渲染的视口数量。

返回值： 视口数量。

.toJSON() : Object

将灯光阴影序列化为 JSON。

参见：
  • ObjectLoader#parse
返回值： 表示序列化灯光阴影的 JSON 对象。

.updateMatrices( light : Light )

更新相机与阴影的矩阵，由渲染器内部使用。

light

正在渲染阴影的灯光。
