RenderPipeline

此模块负责管理应用中的渲染管线配置。 通常创建此类的单个实例，并用它来定义 渲染管线的输出以及后期处理效果链。

注意：此模块只能与 WebGPURenderer 一起使用。

代码示例

const renderPipeline = new RenderPipeline( renderer );
const scenePass = pass( scene, camera );
renderPipeline.outputNode = scenePass;

构造函数

new RenderPipeline( renderer : Renderer, outputNode : Node.<vec4> )

构造一个新的渲染管线管理模块。

renderer

渲染器的引用。
outputNode

可选的输出节点。

属性

.context : Object (readonly)

返回渲染管线栈的当前上下文。

.needsUpdate : Node.<vec4>

当输出节点发生变化时，必须设为 true。

.outputColorTransform : boolean

是否应启用默认的输出色调映射与颜色 空间转换。

默认启用，但对于期望在色调映射和颜色 空间转换之后执行的效果，必须将其禁用。典型例子是 FXAA， 它需要 sRGB 输入。

设为 false 时，应用必须通过 RenderOutputNode 自行控制输出转换。

const outputPass = renderOutput( scenePass );

.outputNode : Node.<vec4>

定义渲染管线最终输出的节点。 通常是效果节点链中的 最后一个节点。

.renderer : Renderer

渲染器的引用。

方法

.dispose()

释放内部资源。

.render()

当使用 RenderPipeline 应用渲染管线与后期处理效果时， 应用必须在动画循环中使用此版本的 render() （而不是渲染器上的那个）。

.renderAsync() : Promise (async)

当使用 RenderPipeline 应用渲染管线与后期处理效果时， 应用必须在动画循环中使用此版本的 renderAsync() （而不是渲染器上的那个）。

已弃用： 是
返回值： 在渲染完成后 resolve 的 Promise。
