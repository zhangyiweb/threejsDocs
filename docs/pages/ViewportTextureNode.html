ViewportTextureNode

一种特殊的纹理节点，将当前视口数据表示为纹理。该模块通过复制操作 从当前绑定的帧缓冲中提取数据，因此无需额外的渲染通道即可生成纹理数据 （有利于性能）。ViewportTextureNode 可用作折射或透射材质等多种效果的输入。

构造函数

new ViewportTextureNode( uvNode : Node, levelNode : Node, framebufferTexture : Texture )

构造一个新的视口纹理节点。

uvNode

uv 节点。

默认值为 screenUV。
levelNode

level 节点。

默认值为 null。
framebufferTexture

保存视口数据的帧缓冲纹理。若未提供，将自动创建帧缓冲纹理。

默认值为 null。

属性

.defaultFramebuffer : FramebufferTexture

参考帧缓冲纹理。用于为当前渲染目标存储帧缓冲纹理。 若渲染目标发生变化，将自动创建新的帧缓冲纹理。

默认值为 null。

.generateMipmaps : boolean

是否生成 mipmap。

默认值为 false。

.isOutputTextureNode : boolean (readonly)

此标志可用于类型测试。

默认值为 true。

.updateBeforeType : string

updateBeforeType 设置为 NodeUpdateType.RENDER，因为该节点应在每次渲染调用时 提取当前绑定帧缓冲的内容。

默认值为 'render'。

重写： TextureNode#updateBeforeType

方法

.getTextureForReference( reference : RenderTarget | CanvasTarget ) : Texture

此方法返回给定渲染目标或画布目标引用对应的纹理。

为避免渲染错误，ViewportTextureNode 必须为不同的渲染上下文使用唯一的帧缓冲纹理。

reference

渲染目标或画布目标引用。

默认值为 null。
返回值： 帧缓冲纹理。
