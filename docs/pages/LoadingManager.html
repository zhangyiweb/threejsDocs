LoadingManager

处理并跟踪已加载与待加载的数据。若不手动提供， 会创建一个默认的全局实例供加载器使用。

一般情况下这已足够，但有时使用单独的加载管理器会很有用—— 例如当你希望为对象和纹理显示不同的加载进度条时。

代码示例

const manager = new THREE.LoadingManager();
manager.onLoad = () => console.log( 'Loading complete!' );
const loader1 = new OBJLoader( manager );
const loader2 = new ColladaLoader( manager );

构造函数

new LoadingManager( onLoad : function, onProgress : function, onError : function )

创建一个新的加载管理器。

onLoad

所有项目加载完成时执行。
onProgress

单个项目加载完成时执行。
onError

发生错误时执行。

属性

.abortController : AbortController

用于中止使用此管理器的加载器中正在进行的请求。

.onError : function | undefined

发生错误时执行。

默认值为 undefined。

.onLoad : function | undefined

所有项目加载完成时执行。

默认值为 undefined。

.onProgress : function | undefined

单个项目加载完成时执行。

默认值为 undefined。

.onStart : function | undefined

项目开始加载时执行。

默认值为 undefined。

方法

.abort() : LoadingManager

可用于中止使用此管理器的加载器中正在进行的加载请求。 仅当加载器实现了 Loader#abort，且浏览器支持 AbortSignal.any() 时，中止才会生效。

返回值： 此加载管理器的引用。

.addHandler( regex : string, loader : Loader ) : LoadingManager

使用给定正则表达式注册加载器。可用于定义应使用哪个加载器来加载特定文件。 典型用例是覆盖纹理的默认加载器。

// add handler for TGA textures
manager.addHandler( /\.tga$/i, new TGALoader() );
regex

正则表达式。
loader

应处理匹配情况的加载器。
返回值： 此加载管理器的引用。

.getHandler( file : string ) : Loader

可用于检索给定文件路径已注册的加载器。

file

文件路径。
返回值： 已注册的加载器。若未找到加载器则返回 null。

.itemEnd( url : string )

使用此管理器的任何加载器在结束加载某个项目时应调用此方法。

url

已加载项目的 URL。

.itemError( url : string )

使用此管理器的任何加载器在加载某个项目时遇到错误应调用此方法。

url

产生错误的项目的 URL。

.itemStart( url : string )

使用此管理器的任何加载器在开始加载某个项目时应调用此方法。

url

要加载的 URL。

.removeHandler( regex : string ) : LoadingManager

移除给定正则表达式对应的加载器。

regex

正则表达式。
返回值： 此加载管理器的引用。

.resolveURL( url : string ) : string

给定一个 URL，使用 URL 修饰回调（若有）并返回解析后的 URL。 若未设置 URL 修饰器，则返回原始 URL。

url

要加载的 URL。
返回值： 解析后的 URL。

.setURLModifier( transform : function ) : LoadingManager

若提供回调，则在发送请求之前，每个资源 URL 都会传给该回调。 回调可以返回原始 URL，或返回新 URL 以覆盖加载行为。 此行为可用于从 .ZIP 文件、拖放 API 和 Data URI 加载资源。

const blobs = {'fish.gltf': blob1, 'diffuse.png': blob2, 'normal.png': blob3};
const manager = new THREE.LoadingManager();
// Initialize loading manager with URL callback.
const objectURLs = [];
manager.setURLModifier( ( url ) => {
	url = URL.createObjectURL( blobs[ url ] );
	objectURLs.push( url );
	return url;
} );
// Load as usual, then revoke the blob URLs.
const loader = new GLTFLoader( manager );
loader.load( 'fish.gltf', (gltf) => {
	scene.add( gltf.scene );
	objectURLs.forEach( ( url ) => URL.revokeObjectURL( url ) );
} );
transform

URL 修饰回调。接收一个 URL，必须返回解析后的 URL。
返回值： 此加载管理器的引用。
