TimestampQueryPool

时间戳查询池的抽象基类。

构造函数

new TimestampQueryPool( maxQueries : number ) (abstract)

创建一个新的时间戳查询池。

maxQueries

此池可容纳的最大查询数量。

默认值为 256。

属性

.currentQueryIndex : number

目前已分配的查询数量。

默认值为 0。

.frames : Array.<number>

存储所有时间戳帧。

.isDisposed : boolean

该池是否已被释放。

默认值为 false。

.lastValue : number

到下一次更新为止的总帧时长。

默认值为 0。

.maxQueries : number

此池可容纳的最大查询数量。

默认值为 256。

.pendingResolve : boolean | Promise.<number>

此属性用于避免多个并发的解析操作。 WebGL 后端将其用作布尔标志。在 WebGPU 上下文中，它保存 当前解析操作的 Promise。

默认值为 false。

.queryOffsets : Map.<string, number>

跟踪不同上下文的偏移量。

.timestamps : Map.<string, number>

存储每个渲染上下文的最新时间戳。

.trackTimestamp : boolean

是否跟踪时间戳。

默认值为 true。

方法

.allocateQueriesForContext( uid : string, frameId : number ) : number (abstract)

为指定的 uid 分配查询。

uid

渲染上下文的唯一标识符。
frameId

当前帧标识符。

.dispose() (abstract)

释放查询池。

.getTimestamp( uid : string ) : number

返回给定渲染上下文的时间戳。

uid

渲染上下文的唯一标识符。
返回值： 时间戳；若不可用则为 undefined。

.getTimestampFrames() : Array.<number>

返回所有时间戳帧。

返回值： 时间戳帧。

.hasTimestampQuery( uid : string ) : boolean

返回给定渲染上下文是否有可用的时间戳。

uid

渲染上下文的唯一标识符。
返回值： 若有可用时间戳则返回 true，否则返回 false。

.resolveQueriesAsync() : Promise.<number> | number (async, abstract)

解析所有时间戳并返回数据（或对其进行处理）。

返回值： 解析后的时间戳值。
