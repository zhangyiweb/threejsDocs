WebGLRenderer

此渲染器使用 WebGL 2 来显示场景。

自 r163 起不再支持 WebGL 1。

构造函数

new WebGLRenderer( parameters : WebGLRenderer~Options )

构造一个新的 WebGL 渲染器。

parameters

配置参数。

属性

.autoClear : boolean

渲染器在渲染一帧之前是否应自动清除其输出。

默认值为 true。

.autoClearColor : boolean

当 WebGLRenderer#autoClear 设为 true 时，渲染器是否应清除颜色缓冲区。

默认值为 true。

.autoClearDepth : boolean

当 WebGLRenderer#autoClear 设为 true 时，渲染器是否应清除深度缓冲区。

默认值为 true。

.autoClearStencil : boolean

当 WebGLRenderer#autoClear 设为 true 时，渲染器是否应清除模板缓冲区。

默认值为 true。

.capabilities : WebGLRenderer~Capabilities

保存当前渲染上下文能力的详细信息。

.clippingPlanes : Array.<Plane>

用户定义的裁剪平面，以世界空间指定。这些平面全局生效。空间中与平面点积为负的点将被裁切。

.coordinateSystem : WebGLCoordinateSystem | WebGPUCoordinateSystem (readonly)

定义渲染器的坐标系。

在 WebGLRenderer 中，该值始终为 WebGLCoordinateSystem。

默认值为 WebGLCoordinateSystem。

.debug : Object

包含调试配置设置的对象。

  • checkShaderErrors：若为 true，则在编译与链接过程中检查材质着色器程序是否有错误。在生产环境中为提升性能可禁用此检查。强烈建议在开发期间保持启用。若着色器未能编译并链接，将无法工作，关联材质也不会渲染。
  • onShaderError(gl, program, glVertexShader,glFragmentShader)：可用于自定义错误报告的回调函数。回调会收到 WebGL 上下文、一个 WebGLProgram 实例，以及分别表示顶点与片元着色器的两个 WebGLShader 实例。指定自定义函数会禁用默认错误报告。

.domElement : HTMLCanvasElement | OffscreenCanvas

渲染器绘制输出的画布。若未预先提供，渲染器会在构造函数中自动创建；只需将其添加到页面中，例如：

document.body.appendChild( renderer.domElement );

.extensions : Object

提供用于获取和检测 WebGL 扩展的方法。

  • get(extensionName:string)：用于检查是否支持某 WebGL 扩展，若可用则返回扩展对象。
  • has(extensionName:string)：若支持该扩展则返回 true。

.info : WebGLRenderer~Info

保存关于 GPU 内存与渲染过程的一系列统计信息。可用于调试与监控。

默认情况下，这些数据会在每次渲染调用时重置；但当每帧有多次渲染通道时（例如使用后处理），可能更适合用自定义方式重置。首先将 autoReset 设为 false。

renderer.info.autoReset = false;

每当完成单帧渲染后调用 reset()。

renderer.info.reset();

.isWebGLRenderer : boolean (readonly)

此标志可用于类型检测。

默认值为 true。

.localClippingEnabled : boolean

渲染器是否遵从对象级裁剪平面。

默认值为 false。

.outputColorSpace : SRGBColorSpace | LinearSRGBColorSpace

定义渲染器的输出色彩空间。

默认值为 SRGBColorSpace。

.properties : Object

用于跟踪其他对象（如原生 WebGL 对象）的属性。

.renderLists : Object

管理渲染器的渲染列表。

.shadowMap : WebGLRenderer~ShadowMap

用于管理阴影的接口。

.sortObjects : boolean

渲染器是否应对对象排序。

注意：排序用于尝试正确渲染具有一定透明度的对象。从定义上讲，对象排序并非在所有情况下都有效。根据应用需求，可能需要关闭排序，并使用其他方式处理透明度渲染，例如手动确定每个对象的渲染顺序。

默认值为 true。

.state : Object

用于管理 WebGL 状态的接口。

.toneMapping : NoToneMapping | LinearToneMapping | ReinhardToneMapping | CineonToneMapping | ACESFilmicToneMapping | CustomToneMapping | AgXToneMapping | NeutralToneMapping

渲染器的色调映射技术。

默认值为 NoToneMapping。

.toneMappingExposure : number

色调映射的曝光级别。

默认值为 1。

.transmissionResolutionScale : number

透射渲染目标的归一化分辨率缩放，以视口尺寸的百分比衡量。使用 MeshPhysicalMaterial#transmission 时，降低该值可显著提升性能。

默认值为 1。

.xr : WebXRManager

对 XR 管理器的引用。

方法

.clear( color : boolean, depth : boolean, stencil : boolean )

指示渲染器清除其颜色、深度或模板绘制缓冲区。此方法会将缓冲区初始化为当前清除色值。

color

是否应清除颜色缓冲区。

默认值为 true。
depth

是否应清除深度缓冲区。

默认值为 true。
stencil

是否应清除模板缓冲区。

默认值为 true。

.clearColor()

清除颜色缓冲区。等价于调用 renderer.clear( true, false, false )。

.clearDepth()

清除深度缓冲区。等价于调用 renderer.clear( false, true, false )。

.clearStencil()

清除模板缓冲区。等价于调用 renderer.clear( false, false, true )。

.compile( scene : Object3D, camera : Camera, targetScene : Scene ) : Set.<Material>

使用相机编译场景中的所有材质。可用于在首次渲染前预编译着色器。若要将 3D 对象添加到已有场景，可使用第三个可选参数指定目标场景。

注意：调用此方法前必须配置好（目标）场景的光照与环境。

scene

要预编译的场景或其他类型的 3D 对象。
camera

相机。
targetScene

目标场景。

默认值为 null。
返回值： 预编译的材质。

.compileAsync( scene : Object3D, camera : Camera, targetScene : Scene ) : Promise (async)

WebGLRenderer#compile 的异步版本。

此方法使用 KHR_parallel_shader_compile WebGL 扩展。因此，建议尽可能使用此版本的 compile()。

scene

要预编译的场景或其他类型的 3D 对象。
camera

相机。
targetScene

目标场景。

默认值为 null。
返回值： 当给定场景可在不被着色器编译不必要阻塞的情况下渲染时解析的 Promise。

默认值为 0。

将给定源纹理的数据复制到目标纹理。

当使用渲染目标纹理作为 srcTexture 和 dstTexture 时，必须确保两个渲染目标都已通过 WebGLRenderer#initRenderTarget 初始化。

srcTexture

描述源区域的包围盒。可以是二维或三维。

默认值为 null。

表示目标区域原点的向量。可以是二维或三维。

默认值为 null。
srcLevel

要复制的源 mipmap 级别。

默认值为 0。
dstLevel

目标 mipmap 级别。

默认值为 0。

.dispose()

.forceContextRestore()

模拟 WebGL 上下文恢复。需要支持 WEBGL_lose_context 扩展。

.render( scene : Object3D, camera : Camera )

使用给定相机渲染给定场景（或其他类型的 3D 对象）。

渲染会输出到先前通过 WebGLRenderer#setRenderTarget 指定的渲染目标，或像往常一样输出到画布。

默认情况下，渲染缓冲区会在渲染前清除；可将属性 autoClear 设为 false 以阻止清除。若只想阻止清除某些缓冲区，可将 autoClearColor、autoClearDepth 或 autoClearStencil 设为 false。若要强制清除，请使用 WebGLRenderer#clear。

scene

要渲染的场景。
camera

相机。

.resetState()

可用于重置内部 WebGL 状态。此方法主要适用于在多个 WebGL 库之间共享同一 WebGL 上下文的应用。

.setAnimationLoop( callback : onAnimationCallback )

此方法允许一次性通过指定宽度、高度和像素比来定义绘图缓冲区尺寸。绘图缓冲区尺寸按以下公式计算：

size.x = width * pixelRatio;
size.y = height * pixelRatio;
