Timer

该类是 Clock 的替代方案，采用不同的 API 设计与行为。 其目标是避免 Clock 随时间推移暴露出的概念性缺陷。

  • Timer 具有 update() 方法用于更新内部状态，因此可在每个模拟步骤中多次调用 getDelta() 和 getElapsed() 而不会得到不同值。
  • 该类可利用 Page Visibility API，在应用处于非活动状态（例如切换标签页或浏览器隐藏）时避免出现过大的时间增量。

代码示例

const timer = new Timer();
timer.connect( document ); // use Page Visibility API

构造函数

new Timer()

构造一个新的计时器。

方法

.connect( document : Document )

将计时器连接到给定文档。调用此方法并非使用计时器的必要条件，但可启用 Page Visibility API 以避免出现过大的时间增量。

document

文档对象。

.disconnect()

将计时器与 DOM 断开连接，并禁用 Page Visibility API 的使用。

.dispose()

释放所有内部资源。当不再需要计时器实例时通常应调用此方法。

.getDelta() : number

返回时间增量（秒）。

返回值： 时间增量（秒）。

.getElapsed() : number

返回经过的时间（秒）。

返回值： 经过的时间（秒）。

.getTimescale() : number

返回时间缩放因子。

返回值： 时间缩放因子。

.reset() : Timer

重置当前模拟步骤的时间计算。

返回值： 对该计时器的引用。

.setTimescale( timescale : number ) : Timer

设置给定的时间缩放因子，用于在 update() 中缩放时间增量计算。

timescale

要设置的时间缩放因子。
返回值： 对该计时器的引用。

.update( timestamp : number ) : Timer

更新计时器的内部状态。每个模拟步骤应调用一次此方法，并在查询计时器（例如通过 getDelta()）之前调用。

timestamp

当前时间（毫秒）。可从 requestAnimationFrame 回调参数获取。若未提供，将使用 performance.now 确定当前时间。
返回值： 对该计时器的引用。
