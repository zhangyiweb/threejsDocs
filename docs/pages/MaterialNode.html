MaterialNode

此类应简化对材质属性的节点访问。 它内部使用引用节点，以确保材质属性的更改 能自动反映到预定义的 TSL 对象中， 例如 materialColor。

构造函数

new MaterialNode( scope : string )

构造一个新的材质节点。

scope

作用域定义了节点所引用的材质属性类型。

属性

.scope : string

作用域定义了节点所引用的材质属性。

方法

.getCache( property : string, type : string ) : MaterialReferenceNode

返回给定属性和类型的缓存引用节点。

property

材质属性的名称。
type

属性的 uniform 类型。
返回值： 表示属性访问的材质引用节点。

.getColor( property : string ) : MaterialReferenceNode.<color>

返回给定属性名称的颜色类型材质引用节点。

property

材质属性的名称。
返回值： 表示属性访问的材质引用节点。

.getFloat( property : string ) : MaterialReferenceNode.<float>

返回给定属性名称的浮点类型材质引用节点。

property

材质属性的名称。
返回值： 表示属性访问的材质引用节点。

.getTexture( property : string ) : MaterialReferenceNode

返回给定属性名称的纹理类型材质引用节点。

property

材质属性的名称。
返回值： 表示属性访问的材质引用节点。

.setup( builder : NodeBuilder ) : Node

节点的设置取决于所选的作用域。多个材质属性如果在逻辑上属于一起， 可能会被组合到单个节点组合中。

builder

当前的节点构建器。
重写： Node#setup
返回值： 表示所选作用域的节点。
