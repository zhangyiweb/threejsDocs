ShaderMaterial

使用自定义着色器渲染的材质。着色器是用 GLSL 编写的小型程序， 在 GPU 上运行。如果你需要实现内置材质未包含的效果， 可能需要使用自定义着色器。

使用 ShaderMaterial 时需注意以下几点：

  • ShaderMaterial 只能与 WebGLRenderer 一起使用。
  • 内置的 attribute 和 uniform 会与你的代码一起传递给着色器。如果 你不希望如此，请改用 RawShaderMaterial。
  • 你可以使用指令 #pragma unroll_loop_start 和 #pragma unroll_loop_end ，以便着色器预处理器展开 GLSL 中的 for 循环。该指令必须 紧挨着写在循环上方。循环格式必须符合既定标准。
    • 循环必须是规范化的。
    • 循环变量必须为 i。
    • 值 UNROLLED_LOOP_INDEX 将被替换为给定迭代中 i 的显式值，并可在预处理器 语句中使用。

代码示例

const material = new THREE.ShaderMaterial( {
	uniforms: {
		time: { value: 1.0 },
		resolution: { value: new THREE.Vector2() }
	},
	vertexShader: document.getElementById( 'vertexShader' ).textContent,
	fragmentShader: document.getElementById( 'fragmentShader' ).textContent
} );

构造函数

new ShaderMaterial( parameters : Object )

构造一个新的着色器材质。

parameters

包含一个或多个属性的对象， 用于定义材质的外观。材质的任何属性 （包括从继承材质中获得的任何属性）都可以传入 此处。颜色值可以传入 Color#set 所接受的任意类型的值。

属性

.clipping : boolean

定义此材质是否支持裁剪；设为 true 时，渲染器会 传递 clippingPlanes uniform。

默认值为 false。

.defaultAttributeValues : Object

当被渲染的几何体不包含这些 attribute，但材质 需要它们时，这些默认值将被传递给着色器。这样 可避免缓冲区数据缺失时出错。

  • color: [ 1, 1, 1 ]
  • uv: [ 0, 0 ]
  • uv1: [ 0, 0 ]

.defines : Object

使用 #define 指令在顶点着色器和片元着色器的 GLSL 代码中定义自定义常量；每一对键/值 都会生成一条指令。

defines: {
	FOO: 15,
	BAR: true
}

会生成以下行：

#define FOO 15
#define BAR true

.extensions : Object

此对象用于启用某些 WebGL 2 扩展。

  • clipCullDistance: 设为 true 以使用顶点着色器裁剪
  • multiDraw: 设为 true 以使用顶点着色器 multi_draw / 启用 gl_DrawID

.fog : boolean

定义材质颜色是否受全局雾效设置影响；设为 true 时会将雾效相关 uniform 传递给着色器。

将此属性设为 true 需要定义雾效相关 uniform。建议 使用 UniformsUtils.merge() 将自定义着色器 uniform 与预定义的雾效 uniform 合并。

const material = new ShaderMaterial( {
    uniforms: UniformsUtils.merge( [ UniformsLib[ 'fog' ], shaderUniforms ] );
    vertexShader: vertexShader,
    fragmentShader: fragmentShader,
    fog: true
} );

默认值为 false。

.forceSinglePass : boolean

已重写，默认设为 true。

默认值为 true。

重写： Material#forceSinglePass

.fragmentShader : string

片元着色器的 GLSL 代码。这是着色器的实际代码。

.glslVersion : GLSL1 | GLSL3

定义自定义着色器代码的 GLSL 版本。

默认值为 null。

.index0AttributeName : string | undefined

若已设置，会调用 gl.bindAttribLocation 将通用顶点索引绑定到 attribute 变量。

默认值为 undefined。

.isShaderMaterial : boolean (readonly)

此标志可用于类型检测。

默认值为 true。

.lights : boolean

定义此材质是否使用光照；设为 true 时会将与光照相关的 uniform 数据传递给此着色器。

默认值为 false。

.linewidth : number

控制线条的粗细。

WebGL 和 WebGPU 会忽略此设置，始终以 一个像素的宽度渲染线元。

默认值为 1。

.uniforms : Object

形如以下结构的对象：

{
	"uniform1": { value: 1.0 },
	"uniform2": { value: 2 }
}

用于指定要传递给着色器代码的 uniform；键为 uniform 名称，值为如下形式的定义

{
	value: 1.0
}

其中 value 是该 uniform 的值。名称必须与 GLSL 代码中 定义的 uniform 名称匹配。请注意，uniform 会在 每一帧刷新，因此更新 uniform 的值会立即 更新 GLSL 代码可用的值。

.uniformsGroups : Array.<UniformsGroup>

保存用于配置 UBO 的 uniforms 组的数组。

.uniformsNeedUpdate : boolean

可用于在 Object3D#onBeforeRender 中更改 uniform 时强制更新 uniform。

默认值为 false。

.vertexShader : string

顶点着色器的 GLSL 代码。这是着色器的实际代码。

.wireframe : boolean

将几何体渲染为线框。

默认值为 false。

.wireframeLinewidth : number

控制线框的粗细。

WebGL 和 WebGPU 会忽略此属性，始终渲染 1 像素宽的线条。

默认值为 1。

方法

.fromJSON( json : Object, textures : Object.<string, Texture> ) : ShaderMaterial

从给定的 JSON 反序列化材质。

json

保存已序列化材质的 JSON。
textures

保存材质所引用纹理的字典。
重写： Material#fromJSON
返回值： 此材质的引用。

类型定义

.Shader

此类型表示存储和运行着色器代码所需的字段。

name
string

着色器名称。
uniforms
Object.<string, Uniform>

着色器的 uniforms。
defines
Object.<string, any>

着色器的 defines。
vertexShader
string

顶点着色器代码。
fragmentShader
string

片元着色器代码。
