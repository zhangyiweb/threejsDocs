LDrawLoader

LDraw 格式的加载器。

LDraw（LEGO Draw）是一种开放格式规范， 用于描述 LEGO 及其他积木类 3D 模型。

LDraw 资源（通常是扩展名为 .ldr、.dat 或 .txt 的文本文件）可以仅描述单个积木零件， 也可以描述整个模型。对于模型，LDraw 文件可以引用其他 LDraw 文件，这些文件会从通过 setPartsLibraryPath 设置的库路径加载。通常需要下载 LDraw 官方零件库， 解压到某个文件夹，并将 setPartsLibraryPath 指向该文件夹。

库零件会在子文件夹 'parts'、'p' 和 'models' 中通过试错方式加载。这些文件访问方式 对 Web 环境并不理想，因此提供了一个脚本工具，可将 LDraw 文件及其所有依赖项 打包为单个文件，加载速度更快。参见「Packing LDraw models」章节。LDrawLoader 示例 加载了多个打包文件。由于体积较大，官方零件库未包含在内。

LDrawLoader 支持以下扩展：

  • !COLOUR：颜色和表面处理声明。
  • BFC：背面剔除规范。
  • !CATEGORY：模型/零件类别声明。
  • !KEYWORDS：模型/零件关键词声明。

代码示例

const loader = new LDrawLoader();
loader.setConditionalLineMaterial( LDrawConditionalLineMaterial ); // the type of line material depends on the used renderer
const object = await loader.loadAsync( 'models/ldraw/officialLibrary/models/car.ldr_Packed.mpd' );
scene.add( object );

导入

LDrawLoader 是一个插件，必须显式导入，参见 Installation#Addons。

import { LDrawLoader } from 'three/addons/loaders/LDrawLoader.js';

构造函数

new LDrawLoader( manager : LoadingManager )

构造一个新的 LDraw 加载器。

manager

加载管理器。

方法

.addDefaultMaterials() : LDrawLoader

使用默认材质初始化加载器。

返回值： 对此加载器的引用。

.addMaterial( material : Material ) : LDrawLoader

向加载器的材质库添加单个材质。

material

要添加的材质。
返回值： 对此加载器的引用。

.addMaterials( materials : Array.<Material> ) : LDrawLoader

向加载器的材质库添加材质列表。

materials

要添加的材质。
返回值： 对此加载器的引用。

.clearMaterials() : LDrawLoader

清空加载器的材质库。

返回值： 对此加载器的引用。

.getMainEdgeMaterial() : Material

返回 LDraw 主边缘颜色的材质。

返回值： 材质。如果未找到材质则返回 null。

.getMainMaterial() : Material

返回 LDraw 主颜色的材质。

对于已加载的 LDraw 资源，返回与主颜色代码关联的材质。 此方法可用于修改暴露主材质的模型或零件。

主颜色代码是为 LDraw 零件着色的标准方式。三角形为 '16'， 边缘为 '24'。通常完整模型不会暴露主颜色（即顶层没有零件 使用代码 '16'，因为它们会被分配其他特定颜色）。而 LDraw 零件文件则会暴露代码 '16' 以供着色，并可能包含其他固定颜色。

返回值： 材质。如果未找到材质则返回 null。

.getMaterial( colorCode : string ) : Material

返回给定颜色代码对应的材质。

colorCode

颜色代码。
返回值： 材质。如果未找到材质则返回 null。

.load( url : string, onLoad : function, onProgress : onProgressCallback, onError : onErrorCallback )

从指定 URL 开始加载，并将加载的 LDraw 资源 传递给 onLoad() 回调。

url

要加载的文件路径/URL，也可以是 data URI。
onLoad

加载完成时执行。
onProgress

加载进行中时执行。
onError

发生错误时执行。
重写： Loader#load

.parse( text : string, onLoad : function, onError : onErrorCallback )

解析给定的 LDraw 数据并返回生成的组。

text

作为字符串的原始 VRML 数据。
onLoad

加载/解析完成时执行。
onError

发生错误时执行。
重写： Loader#parse

.preloadMaterials( url : string ) : Promise (async)

此异步方法从单个 LDraw 文件预加载材质。在官方 零件库中有一个特殊文件会始终首先加载（LDConfig.ldr）， 其中包含所有标准颜色代码。此方法适用于未打包的文件， 例如在编辑器中预加载材质并按需加载零件的场景。

url

LDraw 材质资源的路径。
返回值： 预加载完成时 resolve 的 Promise。

.setConditionalLineMaterial( type : LDrawConditionalLineMaterial.constructor | LDrawConditionalLineNodeMaterial.constructor ) : LDrawLoader

设置条件线材质类型，其取决于所使用的渲染器。 使用 WebGLRenderer 时请使用 LDrawConditionalLineMaterial， 使用 WebGPURenderer 时请使用 LDrawConditionalLineNodeMaterial。

type

条件线材质类型。
返回值： 对此加载器的引用。

.setFileMap( fileMap : Object.<string, string> ) : LDrawLoader

设置将引用的库文件名映射到新文件名的映射表。 如果未指定 fileMap（默认情况），库零件会在子文件夹 'parts'、'p' 和 'models' 中通过试错方式访问。

fileMap

要设置的文件映射。
返回值： 对此加载器的引用。

.setMaterials( materials : Array.<Material> ) : LDrawLoader

设置加载器的材质库。此方法会清除现有的 材质定义。

materials

要设置的材质。
返回值： 对此加载器的引用。

.setPartsLibraryPath( path : string ) : LDrawLoader

除非要加载的模型不引用库零件（通常是将所有零件 打包在单个文件中的模型），否则必须在调用 load() 之前调用此方法。

path

用于加载引用零件的库零件文件路径。 这与 Loader.setPath 不同，后者表示加载主资源的路径。
返回值： 对此加载器的引用。
