XRControllerModelFactory

用于为 WebXR 控制器创建控制器模型，可作为视觉表示添加到场景中。XRControllerModelFactory 会自动获取与用户手持设备尽可能匹配的控制器模型。模型应附加到 getControllerGrip 返回的对象上，以匹配手持设备的方向。

本模块依赖第三方库 motion-controllers。

代码示例

const controllerModelFactory = new XRControllerModelFactory();
const controllerGrip = renderer.xr.getControllerGrip( 0 );
controllerGrip.add( controllerModelFactory.createControllerModel( controllerGrip ) );
scene.add( controllerGrip );

导入

XRControllerModelFactory 是一个插件，必须显式导入，参见 Installation#Addons。

import { XRControllerModelFactory } from 'three/addons/webxr/XRControllerModelFactory.js';

构造函数

new XRControllerModelFactory( gltfLoader : GLTFLoader, onLoad : function )

构造一个新的 XR 控制器模型工厂。

gltfLoader

用于加载控制器模型的 glTF 加载器。

默认值为 null。
onLoad

控制器模型加载完成后执行的回调。

默认值为 null。

属性

.gltfLoader : GLTFLoader

用于加载控制器模型的 glTF 加载器。

默认值为 null。

.onLoad : function

控制器模型加载完成后执行的回调。

默认值为 null。

.path : string

模型资源库的路径。

方法

.createControllerModel( controller : Group ) : XRControllerModel

为给定 WebXR 控制器创建控制器模型。

controller

控制器。
返回值： XR 控制器模型。

.setPath( path : string ) : XRControllerModelFactory

设置模型资源库的路径。

path

要设置的路径。
返回值： 对本实例的引用。
