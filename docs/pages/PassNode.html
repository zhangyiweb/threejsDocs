PassNode

在后处理上下文中表示一个渲染通道（有时也称 beauty pass）。该通道为给定场景和相机构造渲染结果，并可通过 MRT 提供多个输出以供后续处理。

代码示例

const postProcessing = new RenderPipeline( renderer );
const scenePass = pass( scene, camera );
postProcessing.outputNode = scenePass;

构造函数

new PassNode( scope : 'color' | 'depth', scene : Scene, camera : Camera, options : Object )

构造一个新的通道节点。

scope

通道的作用域。作用域决定节点输出颜色还是深度。
scene

场景的引用。
camera

相机的引用。
options

内部渲染目标的选项。

属性

.camera : Camera

相机的引用。

.contextNode : ContextNode | null

通道的可选全局上下文。

.global : boolean

此标志用于全局缓存。

默认值为 true。

重写： TempNode#global

.isPassNode : boolean (readonly)

此标志可用于类型检测。

默认值为 true。

.opaque : boolean

通道是否为不透明。

默认值为 true。

.options : Object

内部渲染目标的选项。

.overrideMaterial : Material | null

通道的可选覆盖材质。

.renderTarget : RenderTarget

通道的渲染目标。

.scene : Scene

场景的引用。

.scope : 'color' | 'depth'

通道的作用域。作用域决定节点输出颜色还是深度。

.transparent : boolean

通道是否为透明。

默认值为 false。

.updateBeforeType : string

updateBeforeType 设为 NodeUpdateType.FRAME，因为该节点在其 PassNode#updateBefore 方法中每帧渲染一次场景。

默认值为 'frame'。

重写： TempNode#updateBeforeType

.COLOR : 'color'

.DEPTH : 'depth'

方法

.compileAsync( renderer : Renderer ) : Promise (async)

预编译该通道。

注意：必须在通道配置完成后调用此方法。因此 setMRT() 和 getTextureNode() 等调用必须在预编译之前完成。

renderer

渲染器。
参见：
  • Renderer#compileAsync
返回值： 编译完成时 resolve 的 Promise。

.dispose()

释放内部资源。当节点不再使用时应调用此方法。

重写： TempNode#dispose

.getLayers() : Layers

获取通道当前的图层配置。

返回值： 图层配置。

.getLinearDepthNode( name : string ) : Node

返回此通道的线性深度节点。

name

要获取线性深度节点的输出名称。多数情况下可使用默认的 'depth'，但该参数也支持自定义深度输出。

默认值为 'depth'。
返回值： 线性深度节点。

.getMRT() : MRTNode

返回当前的 MRT 节点。

返回值： 当前的 MRT 节点。

.getPreviousTexture( name : string ) : Texture

返回保存给定输出名称上一帧数据的纹理。

name

要获取纹理的输出名称。
返回值： 保存上一帧数据的纹理。

.getPreviousTextureNode( name : string ) : TextureNode

返回给定输出名称的上一帧纹理节点。

name

要获取上一帧纹理节点的输出名称。

默认值为 'output'。
返回值： 上一帧纹理节点。

.getResolution() : number

获取通道当前的分辨率。

已弃用： 自 r181 起。请改用 `getResolutionScale()`。
返回值： 当前分辨率。值为 1 表示全分辨率。

.getResolutionScale() : number

获取通道当前的分辨率缩放。

返回值： 当前分辨率缩放。值为 1 表示全分辨率。

.getTexture( name : string ) : Texture

返回给定输出名称的纹理。

name

要获取纹理的输出名称。
返回值： 纹理。

.getTextureNode( name : string ) : TextureNode

返回给定输出名称的纹理节点。

name

要获取纹理节点的输出名称。

默认值为 'output'。
返回值： 纹理节点。

.getViewZNode( name : string ) : Node

返回此通道的 viewZ 节点。

name

要获取 viewZ 节点的输出名称。多数情况下可使用默认的 'depth'，但该参数也支持自定义深度输出。

默认值为 'depth'。
返回值： viewZ 节点。

.setLayers( layers : Layers ) : PassNode

设置渲染通道时应使用的图层配置。

layers

要设置的图层对象。
返回值： 对此通道的引用。

.setMRT( mrt : MRTNode ) : PassNode

设置给定的 MRT 节点，为此通道配置 MRT。

mrt

MRT 对象。
返回值： 对此通道的引用。

.setResolution( resolution : number ) : PassNode

设置通道的分辨率。 分辨率是一个与渲染器宽高相乘的因子。

resolution

要设置的分辨率。值为 1 表示全分辨率。
已弃用： 自 r181 起。请改用 `setResolutionScale()`。
返回值： 对此通道的引用。

.setResolutionScale( resolutionScale : number ) : PassNode

设置通道的分辨率缩放。 分辨率缩放是一个与渲染器宽高相乘的因子。

resolutionScale

要设置的分辨率缩放。值为 1 表示全分辨率。
返回值： 对此通道的引用。

.setScissor( x : number | Vector4, y : number, width : number, height : number )

此方法用于定义通道的裁剪矩形。默认情况下，裁剪矩形与通道尺寸保持同步。若要恢复自动尺寸，请以 null 作为唯一参数调用此方法。

x

裁剪框左下角在逻辑像素单位下的水平坐标。 除传入四个参数外，也可传入单个四维向量。
y

裁剪框左下角在逻辑像素单位下的垂直坐标。
width

裁剪框在逻辑像素单位下的宽度。
height

裁剪框在逻辑像素单位下的高度。

.setSize( width : number, height : number )

设置通道渲染目标的尺寸。会考虑像素比。

width

要设置的宽度。
height

要设置的高度。

.setViewport( x : number | Vector4, y : number, width : number, height : number )

此方法用于定义通道的视口。默认情况下，视口与通道尺寸保持同步。若要恢复自动尺寸，请以 null 作为唯一参数调用此方法。

x

视口原点在逻辑像素单位下左下角的水平坐标。
y

视口原点在逻辑像素单位下左下角的垂直坐标。
width

视口在逻辑像素单位下的宽度。
height

视口在逻辑像素单位下的高度。

.toggleTexture( name : string )

切换给定输出名称的当前纹理与上一帧纹理。

name

输出名称。
