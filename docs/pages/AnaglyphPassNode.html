AnaglyphPassNode

使用物理正确的离轴立体投影创建红蓝立体（Anaglyph）效果的渲染通道节点。

此实现使用 CameraUtils.frameCorners() 将立体相机视锥体对齐到虚拟屏幕平面，在平面距离处提供准确的深度感知和零视差。

导入

AnaglyphPassNode 是一个插件，必须显式导入，参见 Installation#Addons。

import { anaglyphPass, AnaglyphAlgorithm, AnaglyphColorMode } from 'three/addons/tsl/display/AnaglyphPassNode.js';

构造函数

new AnaglyphPassNode( scene : Scene, camera : Camera )

构造一个新的红蓝立体通道节点。

scene

要渲染的场景。
camera

用于渲染场景的相机。

属性

.algorithm : string

获取当前的红蓝立体算法。

.algorithm : string

设置红蓝立体算法。

.colorMode : string

获取当前的颜色模式。

.colorMode : string

设置颜色模式。

.eyeSep : number

瞳距（双眼间距），以世界单位表示。 人类典型瞳距为 0.064 米（64 毫米）。

默认值为 0.064。

.isAnaglyphPassNode : boolean (readonly)

此标志可用于类型测试。

默认值为 true。

.planeDistance : number

从观察者到虚拟屏幕平面的距离（世界单位），在该距离处产生零视差（屏幕深度）。 处于此距离的物体看起来位于屏幕表面。 更近的物体看起来在屏幕前方（负视差）。 更远的物体看起来在屏幕后方（正视差）。

屏幕尺寸根据相机在此距离处的 FOV 和宽高比计算， 确保立体视图与相机的视场角匹配。

默认值为 0.5。
