BufferAttributeNode

在较早版本的 three.js 中，只能在几何体级别定义属性数据。 借助 BufferAttributeNode，也可以在节点级别完成这一操作。

const geometry = new THREE.PlaneGeometry();
const positionAttribute = geometry.getAttribute( 'position' );
const colors = [];
for ( let i = 0; i < position.count; i ++ ) {
	colors.push( 1, 0, 0 );
}
material.colorNode = bufferAttribute( new THREE.Float32BufferAttribute( colors, 3 ) );

当几何体数据通过计算着色器生成时，这种新方式尤其有用。 下面这行代码将存储缓冲区转换为属性节点。

material.positionNode = positionBuffer.toAttribute();

构造函数

new BufferAttributeNode( value : BufferAttribute | InterleavedBuffer | TypedArray, bufferType : string, bufferStride : number, bufferOffset : number )

构造一个新的缓冲属性节点。

value

属性数据。
bufferType

缓冲区类型（例如 'vec3'）。

默认值为 null。
bufferStride

缓冲区步幅。

默认值为 0。
bufferOffset

缓冲区偏移量。

默认值为 0。

属性

.attribute : BufferAttribute

对缓冲属性的引用。

默认值为 null。

.bufferOffset : number

缓冲区偏移量。

默认值为 0。

.bufferStride : number

缓冲区步幅。

默认值为 0。

.bufferType : string

缓冲区类型（例如 'vec3'）。

默认值为 null。

.global : boolean

BufferAttributeNode 默认将该属性设为 true。

默认值为 true。

重写： InputNode#global

.instanced : boolean

该属性是否为实例化属性。

默认值为 false。

.isBufferNode : boolean (readonly)

该标志可用于类型检测。

默认值为 true。

.usage : number

用法属性。若计划每帧更新属性数据， 请通过 .setUsage() 将其设为 THREE.DynamicDrawUsage。

默认值为 StaticDrawUsage。

方法

.generate( builder : NodeBuilder ) : string

生成缓冲属性节点的代码片段。

builder

当前的节点构建器。
重写： InputNode#generate
返回值： 生成的代码片段。

.generateNodeType( builder : NodeBuilder ) : string

该方法被重写，因为节点类型是从缓冲属性推断得出的。

builder

当前的节点构建器。
重写： InputNode#generateNodeType
返回值： 节点类型。

.getHash( builder : NodeBuilder ) : string

该方法被重写，因为属性数据可能被共享， 因此哈希也应共享。

builder

当前的节点构建器。
重写： InputNode#getHash
返回值： 哈希值。

.getInputType( builder : NodeBuilder ) : string

重写默认实现，返回固定值 'bufferAttribute'。

builder

当前的节点构建器。
重写： InputNode#getInputType
返回值： 输入类型。

.setInstanced( value : boolean ) : BufferAttributeNode

将 instanced 属性设置为给定值。

value

要设置的值。
返回值： 对该节点的引用。

.setUsage( value : number ) : BufferAttributeNode

将 usage 属性设置为给定值。

value

要设置的用法。
返回值： 对该节点的引用。

.setup( builder : NodeBuilder )

根据传入节点的值不同，setup() 的行为也不同。 若未传入 BufferAttribute 实例，该方法会创建内部属性并相应配置。

builder

当前的节点构建器。
重写： InputNode#setup
