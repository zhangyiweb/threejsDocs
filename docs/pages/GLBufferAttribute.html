GLBufferAttribute

缓冲属性的替代版本，可对 VBO 进行更精细的控制。

渲染器不会为这类属性构造 VBO，而是使用构造函数中传入的 VBO，之后也可通过 buffer 属性修改。

最常见的用例是某种 GPGPU 计算介入甚至直接生成相关 VBO 时。

注意：此类仅可与 WebGLRenderer 配合使用。

构造函数

new GLBufferAttribute( buffer : WebGLBuffer, type : number, itemSize : number, elementSize : number, count : number, normalized : boolean )

构造一个新的 GL 缓冲属性。

buffer

原生 WebGL 缓冲区。
type

原生数据类型（例如 gl.FLOAT）。
itemSize

每个顶点的分量数量。
elementSize

给定 type 参数对应的字节大小。
count

VBO 中预期的顶点数量。
normalized

数据是否已归一化。

默认值为 false。

属性

.buffer : WebGLBuffer

原生 WebGL 缓冲区。

.count : number

VBO 中预期的顶点数量。

.elementSize : number

给定 type 参数对应的字节大小。

.isGLBufferAttribute : boolean (readonly)

此标志可用于类型检测。

默认值为 true。

.itemSize : number

每个顶点的分量数量，参见 BufferAttribute#itemSize。

.name : string

缓冲属性的名称。

.needsUpdate : number

标志该属性已更改，应重新发送到 GPU。修改数组值时请将其设为 true。

默认值为 false。

.normalized : boolean

仅适用于整数数据。指示缓冲区中的底层数据如何映射到 GLSL 代码中的值。例如，若 buffer 包含 gl.UNSIGNED_SHORT 数据，且 normalized 为 true，则缓冲区中的 0 - +65535 会映射为 GLSL 属性中的 0.0f - +1.0f。若 normalized 为 false，这些值将原样转换为浮点数，即 65535 变为 65535.0f。

.type : number

原生数据类型。

.version : number

版本号，每次将 needsUpdate 设为 true 时递增。

方法

.setBuffer( buffer : WebGLBuffer ) : BufferAttribute

设置给定的原生 WebGL 缓冲区。

buffer

要设置的缓冲区。
返回值： 对该实例的引用。

.setCount( count : number ) : BufferAttribute

设置计数（VBO 中预期的顶点数量）。

count

计数。
返回值： 对该实例的引用。

.setItemSize( itemSize : number ) : BufferAttribute

设置每个顶点的分量数量。

itemSize

每个顶点的分量数量。
返回值： 对该实例的引用。

.setType( type : number, elementSize : number ) : BufferAttribute

设置给定的原生数据类型和元素大小。

type

原生数据类型（例如 gl.FLOAT）。
elementSize

给定 type 参数对应的字节大小。
返回值： 对该实例的引用。
