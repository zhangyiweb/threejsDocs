UniformArrayNode

与 BufferNode 类似，此模块将类数组数据表示为 uniform 缓冲区。与 BufferNode 不同，它能处理数组中更常见的 数据类型（例如 three.js 基本类型），并自动 管理缓冲区填充。在处理 uniform 缓冲区时应作为首选。

代码示例

const tintColors = uniformArray( [
	new Color( 1, 0, 0 ),
	new Color( 0, 1, 0 ),
	new Color( 0, 0, 1 )
], 'color' );
const redColor = tintColors.element( 0 );

构造函数

new UniformArrayNode( value : Array.<any>, elementType : string )

构造一个新的 uniform 数组节点。

value

保存缓冲区数据的数组。
elementType

缓冲区元素的数据类型。

默认值为 null。

属性

.array : Array.<any>

保存缓冲区数据的数组。与 BufferNode 不同，该数组可以 保存数字基本类型以及 three.js 对象，如向量、矩阵 或颜色。

.elementType : string

数组元素的数据类型。

.isArrayBufferNode : boolean (readonly)

此标志可用于类型测试。

默认值为 true。

.paddedType : string

填充后的类型。Uniform 缓冲区必须遵循特定的缓冲区布局， 因此会计算单独的类型以确保正确的缓冲区大小。

.updateType : string

已重写，因为 uniform 数组节点按每次渲染更新。

默认值为 'render'。

重写： BufferNode#updateType

方法

.element( indexNode : IndexNode ) : UniformArrayElementNode

重写默认的 element() 方法，以基于 UniformArrayNode 提供元素访问。

indexNode

索引节点。

.generateNodeType( builder : NodeBuilder ) : string

此方法已重写，因为节点类型从 UniformArrayNode#paddedType 推断。

builder

当前节点构建器。
重写： BufferNode#generateNodeType
返回值： 节点类型。

.getElementType( builder : NodeBuilder ) : string

数组元素的数据类型。

builder

当前节点构建器。
重写： BufferNode#getElementType
返回值： 元素类型。

.getPaddedType() : string

根据元素类型返回填充后的类型。

返回值： 填充后的类型。

.setup( builder : NodeBuilder ) : null

根据数组数据实现值缓冲区的创建。

builder

对当前节点构建器的引用。
重写： BufferNode#setup

.update( frame : NodeFrame )

更新确保正确地将数组中（复杂）对象的数据 传输到内部、正确填充的值缓冲区。

frame

对当前节点帧的引用。
重写： BufferNode#update
