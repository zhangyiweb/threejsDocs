TRAANode

应用 TRAA（时域重投影抗锯齿）的特殊节点。

参考文献：

  • https://alextardif.com/TAA.html
  • https://www.elopezr.com/temporal-aa-and-the-quest-for-the-holy-trail/

注意：使用 TRAA 时必须禁用 MSAA。

导入

TRAANode 是一个插件，必须显式导入，参见 Installation#Addons。

import { traa } from 'three/addons/tsl/display/TRAANode.js';

构造函数

new TRAANode( beautyNode : TextureNode, depthNode : TextureNode, velocityNode : TextureNode, camera : Camera )

构造一个新的 TRAA 节点。

beautyNode

表示效果输入的纹理节点。
depthNode

表示场景深度的节点。
velocityNode

表示场景速度的节点。
camera

用于渲染场景的相机。

属性

.beautyNode : TextureNode

表示效果输入的纹理节点。

.camera : Camera

用于渲染场景的相机。

.depthNode : TextureNode

表示场景深度的节点。

.depthThreshold : number

当当前深度与上一帧深度的差值超过此阈值时，历史帧被视为无效。

默认值为 0.0005。

.edgeDepthDiff : number

3×3 邻域内的深度差值，用于判定像素是否为边缘。

默认值为 0.001。

.isTRAANode : boolean (readonly)

此标志可用于类型检测。

默认值为 true。

.maxVelocityLength : number

当速度的像素长度接近此值时，历史帧将变为无效。

默认值为 128。

.updateBeforeType : string

updateBeforeType 设为 NodeUpdateType.FRAME，因为该节点在 updateBefore() 中每帧渲染一次效果。

默认值为 'frame'。

重写： TempNode#updateBeforeType

.useSubpixelCorrection : boolean

当速度更接近亚像素级别时，是否降低当前帧的权重。这有助于减少运动下的模糊，但可能引入方形图案伪影。

默认值为 true。

.velocityNode : TextureNode

表示场景速度的节点。

方法

.clearViewOffset()

清除场景相机的视图偏移。

.dispose()

释放内部资源。当效果不再使用时，应调用此方法。

重写： TempNode#dispose

.getTextureNode() : PassTextureNode

以纹理节点的形式返回效果结果。

返回值： 表示效果结果的纹理节点。

.setSize( width : number, height : number )

设置效果的尺寸。

width

效果的宽度。
height

效果的高度。

.setViewOffset( width : number, height : number )

将 TRAA 当前的抖动定义为场景相机的视图偏移。

width

效果的宽度。
height

效果的高度。

.setup( builder : NodeBuilder ) : PassTextureNode

此方法用于设置效果的渲染目标与 TSL 代码。

builder

当前的节点构建器。
重写： TempNode#setup

.updateBefore( frame : NodeFrame )

此方法用于每帧渲染一次效果。

frame

当前的节点帧。
重写： TempNode#updateBefore
