USDComposer

USDComposer 负责根据已解析的 USD 数据完成场景合成。 包括引用解析、变体选择、变换处理，以及构建 Three.js 场景图。

与 USDCParser 输出的 specsByPath 格式配合使用。

构造函数

new USDComposer()

方法

._applyMaterialBinding()

将材质绑定从 prim 路径应用到网格。 用于将引用的几何体合并到带有材质绑定的 prim 时。

._applyTextureOrValue()

用于从着色器属性应用纹理或数值的共享辅助方法。 减少 _applyPreviewSurface 与 _applyOpenPBRSurface 之间的重复代码。

._buildCamera()

根据 Camera 规范构建相机。

._buildGeomPrimitive()

根据 USD 几何图元（Cube、Sphere、Cylinder、Cone、Capsule）构建网格。

._buildHierarchy()

递归构建场景层级。 使用 childrenByPath 索引实现 O(1) 子节点查找，而非 O(n) 遍历。

._buildIndexes()

构建用于高效查找的索引。 在 compose() 期间调用一次，以避免每次查找都进行 O(n) 扫描。

._buildLight()

根据 UsdLux 灯光规范构建灯光。

._buildMesh()

根据 Mesh 规范构建网格。

._colorTemperature()

将开尔文色温转换为 RGB 颜色。 基于 Tanner Helland 的算法。

._computeVertexNormals()

根据索引三角形数据计算逐顶点法线。 在每个共享顶点处累加面积加权的面法线，然后归一化。

._findSingleMesh()

在组的浅层层级中查找单个网格。 仅当网格位于深度 0 或 1（而非深层嵌套）时才返回。 这样可在 Kitchen Set 等复杂层级中保留变换， 同时支持 USDZExporter 往返（Xform > Xform > Mesh 模式）。

._getAttributes()

从属性规范中获取指定路径的属性。

._getBasePath()

从文件路径获取基础路径（目录）。

._getLocalVariantSelections()

从规范的 fields 中提取变体选择。

._getMaterialBindingTarget()

获取材质绑定目标路径，必要时检查变体路径。

._getMaterialPath()

获取网格的材质路径，检查各种绑定来源。

._getReferences() : Array.<string>

从 prim 规范中获取所有引用值。

返回值： 引用字符串数组，例如 "@path@" 或 "@path@<prim>"

._getVariantPaths()

根据变体选择获取父路径的变体路径。

._hasNonIdentityTransform()

检查对象是否具有非单位矩阵的局部变换。

._isDirectChild()

检查路径是否为 parentPath 的直接子路径。

._resolveFilePath()

相对于 basePath 解析文件路径。

._resolveReference( refValue : string, localVariants : Object ) : Group | null

解析 USD 引用并返回合成后的内容。

refValue

引用值，例如 "@./path/to/file.usdc@"
localVariants

要应用的变体选择
返回值： 合成后的内容，或 null

.applyTransform()

将 USD 变换应用到 Three.js 对象。 按 xformOpOrder 进行正确的矩阵合成。 USD 使用行向量约定，Three.js 使用列向量约定。

.compose( parsedData : Object, assets : Object, variantSelections : Object, basePath : string ) : Group

根据已解析的 USD 数据合成 Three.js 场景。

parsedData

来自 USDCParser 或 USDAParser 的数据
assets

被引用资源的字典（specsByPath 或 blob URL）
variantSelections

外部变体选择
basePath

用于解析相对引用的基础路径
返回值： Three.js 场景图
