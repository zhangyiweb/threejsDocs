BatchedMesh

一种支持多绘制批处理渲染的特殊网格版本。当你需要渲染大量 材质相同、但几何体或世界变换不同的对象时，请使用此类。使用 BatchedMesh 有助于减少绘制调用次数，从而提升应用的整体 渲染性能。

代码示例

const box = new THREE.BoxGeometry( 1, 1, 1 );
const sphere = new THREE.SphereGeometry( 1, 12, 12 );
const material = new THREE.MeshBasicMaterial( { color: 0x00ff00 } );
// initialize and add geometries into the batched mesh
const batchedMesh = new BatchedMesh( 10, 5000, 10000, material );
const boxGeometryId = batchedMesh.addGeometry( box );
const sphereGeometryId = batchedMesh.addGeometry( sphere );
// create instances of those geometries
const boxInstancedId1 = batchedMesh.addInstance( boxGeometryId );
const boxInstancedId2 = batchedMesh.addInstance( boxGeometryId );
const sphereInstancedId1 = batchedMesh.addInstance( sphereGeometryId );
const sphereInstancedId2 = batchedMesh.addInstance( sphereGeometryId );
// position the geometries
batchedMesh.setMatrixAt( boxInstancedId1, boxMatrix1 );
batchedMesh.setMatrixAt( boxInstancedId2, boxMatrix2 );
batchedMesh.setMatrixAt( sphereInstancedId1, sphereMatrix1 );
batchedMesh.setMatrixAt( sphereInstancedId2, sphereMatrix2 );
scene.add( batchedMesh );

构造函数

new BatchedMesh( maxInstanceCount : number, maxVertexCount : number, maxIndexCount : number, material : Material | Array.<Material> )

构造一个新的批处理网格。

maxInstanceCount

计划添加并渲染的独立实例的最大数量。
maxVertexCount

所有唯一几何体将使用的最大顶点数。
maxIndexCount

所有唯一几何体将使用的最大索引数。

默认值为 maxVertexCount*2。
material

网格材质。

属性

.boundingBox : Box3

批处理网格的包围盒。可通过 BatchedMesh#computeBoundingBox 计算。

默认值为 null。

.boundingSphere : Sphere

批处理网格的包围球。可通过 BatchedMesh#computeBoundingSphere 计算。

默认值为 null。

.customSort : function

接收一个在渲染前运行的排序函数。该函数接收待排序的实例列表 以及一个相机。列表中的对象包含一个 "z" 字段，可用于按深度排序。

默认值为 null。

.instanceCount : number (readonly)

实例数量。

.isBatchedMesh : boolean (readonly)

此标志可用于类型检测。

默认值为 true。

.maxInstanceCount : number (readonly)

批处理中可存储的独立实例的最大数量。

.perObjectFrustumCulled : boolean

设为 true 时，批处理中的各个对象会进行视锥体裁剪。

默认值为 true。

.sortObjects : boolean

设为 true 时，批处理中的各个对象会被排序，以改善过度绘制相关伪影。 若材质标记为 "transparent"，则对象从后往前渲染；否则 从前往后渲染。

默认值为 true。

.unusedIndexCount : number (readonly)

未使用的索引数量。

.unusedVertexCount : number (readonly)

未使用的顶点数量。

方法

.addGeometry( geometry : BufferGeometry, reservedVertexCount : number, reservedIndexCount : number ) : number

将给定几何体添加到批处理中，并返回与之关联的 几何体 ID，供其他函数使用。

geometry

要添加的几何体。
reservedVertexCount

可选参数，指定为所添加几何体预留的 顶点缓冲区空间量。若计划稍后在该索引处设置一个比原始几何体更大的新几何体，则需要此参数。 默认为给定几何体顶点缓冲区的长度。

默认值为 -1。
reservedIndexCount

可选参数，指定为所添加几何体预留的 索引缓冲区空间量。若计划稍后在该索引处设置一个比原始几何体更大的新几何体，则需要此参数。默认为 给定几何体索引缓冲区的长度。

默认值为 -1。
返回值： 几何体 ID。

.addInstance( geometryId : number ) : number

使用给定 ID 的几何体向批处理添加一个新实例，并返回 一个引用该新实例的新 ID，供其他函数使用。

geometryId

先前通过 BatchedMesh#addGeometry 添加的几何体的 ID。
返回值： 实例 ID。

.computeBoundingBox()

计算包围盒，并更新 BatchedMesh#boundingBox。 包围盒默认不会计算，需要显式调用计算， 否则其为 null。

.computeBoundingSphere()

计算包围球，并更新 BatchedMesh#boundingSphere。 包围球默认不会计算，需要显式调用计算， 否则其为 null。

.deleteGeometry( geometryId : number ) : BatchedMesh

从该批处理中删除由给定 ID 定义的几何体。引用 此几何体的所有实例也会作为副作用被移除。

geometryId

要从批处理中移除的几何体的 ID。
返回值： 对该批处理网格的引用。

.deleteInstance( instanceId : number ) : BatchedMesh

使用给定 ID 从批处理中删除一个已有实例。

instanceId

要从批处理中移除的实例的 ID。
返回值： 对该批处理网格的引用。

.dispose()

释放该实例分配的 GPU 相关资源。当应用中 不再使用该实例时，请调用此方法。

.getBoundingBoxAt( geometryId : number, target : Box3 ) : Box3

返回给定几何体的包围盒。

geometryId

要返回包围盒的几何体的 ID。
target

用于存储方法结果的目标对象。
返回值： 该几何体的包围盒。若未找到给定 ID 的几何体，则返回 null。

.getBoundingSphereAt( geometryId : number, target : Sphere ) : Sphere

返回给定几何体的包围球。

geometryId

要返回包围球的几何体的 ID。
target

用于存储方法结果的目标对象。
返回值： 该几何体的包围球。若未找到给定 ID 的几何体，则返回 null。

.getColorAt( instanceId : number, color : Color | Vector4 ) : Color | Vector4

返回指定实例的颜色。

instanceId

要获取颜色的实例的 ID。
color

用于存储方法结果的目标对象。
返回值： 该实例的颜色。使用 Vector4 还可获取 alpha。

.getGeometryIdAt( instanceId : number ) : number

返回指定实例的几何体 ID。

instanceId

要获取几何体 ID 的实例的 ID。
返回值： 该实例的几何体 ID。

.getGeometryRangeAt( geometryId : number, target : Object ) : Object

获取表示所附加几何体相关三角形子集的范围， 指示起始偏移和数量；若无效则返回 null。

geometryId

要获取范围的几何体的 ID。
target

用于存储方法结果的目标对象。
返回值： 包含范围数据的结果对象。

.getMatrixAt( instanceId : number, matrix : Matrix4 ) : Matrix4

返回指定实例的本地变换矩阵。

instanceId

要获取矩阵的实例的 ID。
matrix

用于存储方法结果的目标对象。
返回值： 该实例的本地变换矩阵。

.getVisibleAt( instanceId : number ) : boolean

返回指定实例的可见性状态。

instanceId

要获取可见性状态的实例的 ID。
返回值： 该实例是否可见。

.optimize() : BatchedMesh

重新打包 BatchedMesh 中的子几何体，以移除先前删除几何体后留下的 未使用空间，从而释放空间以添加新几何体。

返回值： 对该批处理网格的引用。

.setColorAt( instanceId : number, color : Color | Vector4 ) : BatchedMesh

将给定颜色设置到指定实例。

instanceId

要设置颜色的实例的 ID。
color

要设置到该实例的颜色。使用 Vector4 还可定义 alpha。
返回值： 对该批处理网格的引用。

.setCustomSort( func : function ) : BatchedMesh

接收一个在渲染前运行的排序函数。该函数接收待排序的实例列表 以及一个相机。列表中的对象包含一个 "z" 字段，可用于按深度排序。

func

自定义排序函数。
返回值： 对该批处理网格的引用。

.setGeometryAt( geometryId : number, geometry : BufferGeometry ) : number

用提供的几何体替换给定 ID 处的几何体。若没有为几何体预留足够空间， 则抛出错误。调用此方法将更改所有正在渲染该几何体的实例。

geometryId

应被给定几何体替换的几何体的 ID。
geometry

新的几何体。
返回值： 几何体 ID。

.setGeometryIdAt( instanceId : number, geometryId : number ) : BatchedMesh

设置给定索引处实例的几何体 ID。

instanceId

要设置几何体 ID 的实例的 ID。
geometryId

该实例将使用的几何体 ID。
返回值： 对该批处理网格的引用。

.setGeometrySize( maxVertexCount : number, maxIndexCount : number )

将批处理的顶点和索引缓冲区属性的可用空间调整为所提供的大小。 若所提供参数会缩小几何体缓冲区，但几何体属性末尾没有足够的未使用空间， 则抛出错误。

maxVertexCount

所有唯一几何体将使用的最大顶点数（调整到的目标值）。
maxIndexCount

所有唯一几何体将使用的最大索引数（调整到的目标值）。

.setInstanceCount( maxInstanceCount : number )

调整必要的缓冲区以支持所提供的实例数量。 若所提供参数会缩小实例数量，但列表末尾没有足够的 未使用 ID，则抛出错误。

maxInstanceCount

批处理可添加并渲染的独立实例的最大数量。

.setMatrixAt( instanceId : number, matrix : Matrix4 ) : BatchedMesh

将给定本地变换矩阵设置到指定实例。 不支持负缩放矩阵。

instanceId

要设置矩阵的实例的 ID。
matrix

表示单个实例本地变换的 4x4 矩阵。
返回值： 对该批处理网格的引用。

.setVisibleAt( instanceId : number, visible : boolean ) : BatchedMesh

设置实例的可见性。

instanceId

要设置可见性的实例的 ID。
visible

该实例是否可见。
返回值： 对该批处理网格的引用。

.validateGeometryId( geometryId : number )

验证由给定 ID 定义的几何体。

geometryId

要验证的几何体。

.validateInstanceId( instanceId : number )

验证由给定 ID 定义的实例。

instanceId

要验证的实例。
