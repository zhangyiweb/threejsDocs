KTX2Loader

用于加载 KTX 2.0 GPU 纹理容器的加载器。

KTX 2.0 是各种 GPU 纹理格式的容器格式。该加载器支持 Basis Universal GPU 纹理， 可快速转码为多种 GPU 纹理压缩格式。虽然 KTX 2.0 也允许其他硬件特定格式，但此加载器尚未解析它们。

此加载器解析 KTX 2.0 容器并转码为支持的 GPU 压缩纹理格式。 所需的 WASM 转码器和 JS 包装器可从 examples/jsm/libs/basis 目录获取。

此加载器依赖 Web Assembly，旧版浏览器不支持。

参考：

  • KTX specification
  • DFD
  • BasisU HDR

代码示例

const loader = new KTX2Loader();
loader.setTranscoderPath( 'examples/jsm/libs/basis/' );
loader.detectSupport( renderer );
const texture = loader.loadAsync( 'diffuse.ktx2' );

导入

KTX2Loader 是一个插件，必须显式导入，参见 Installation#Addons。

import { KTX2Loader } from 'three/addons/loaders/KTX2Loader.js';

构造函数

new KTX2Loader( manager : LoadingManager )

构造一个新的 KTX2 加载器。

manager

加载管理器。

方法

.detectSupport( renderer : WebGPURenderer | WebGLRenderer ) : KTX2Loader

检测硬件对可用压缩纹理格式的支持，以确定 转码器的输出格式。必须在加载纹理之前调用。

renderer

渲染器。
返回值： 此加载器的引用。

.detectSupportAsync( renderer : WebGPURenderer ) : Promise (async)

KTX2Loader#detectSupport 的异步版本。

renderer

渲染器。
已弃用： 是
返回值： 在检测到支持后解析的 Promise。

.dispose()

释放内部资源。当不再需要此加载器时应调用 此方法。

.load( url : string, onLoad : function, onProgress : onProgressCallback, onError : onErrorCallback )

从指定 URL 开始加载，并将加载完成的 KTX2 纹理传递给 onLoad() 回调。

url

要加载的文件路径/URL，也可以是 data URI。
onLoad

加载完成时执行。
onProgress

加载进行中时执行。
onError

发生错误时执行。
重写： Loader#load

.parse( buffer : ArrayBuffer, onLoad : function, onError : onErrorCallback ) : Promise

解析给定的 KTX2 数据。

buffer

原始 KTX2 数据（ArrayBuffer 格式）。
onLoad

加载/解析完成时执行。
onError

发生错误时执行。
重写： Loader#parse
返回值： 在解析完成后解析的 Promise。

.setTranscoderPath( path : string ) : KTX2Loader

设置转码器路径，可选择从 CDN 设置解码器加载路径。

默认情况下，WASM 转码器和 JS 包装器从 examples/jsm/libs/basis 目录加载。

path

要设置的转码器路径。
返回值： 此加载器的引用。

.setWorkerLimit( workerLimit : number ) : KTX2Loader

设置此实例分配的最大 Web Worker 数量。

workerLimit

Worker 数量限制。
返回值： 此加载器的引用。
