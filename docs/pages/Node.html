Node

所有节点的基类。

构造函数

new Node( nodeType : string )

构造一个新节点。

nodeType

节点类型。

默认值为 null。

属性

.global : boolean

此节点是否为全局节点。该属性与内部节点缓存系统相关。 所有应只声明一次的节点应将此标志设为 true （典型示例为 AttributeNode）。

默认值为 false。

.id : number (readonly)

节点的唯一 ID。

.isNode : boolean (readonly)

此标志可用于类型检测。

默认值为 true。

.name : string

节点的名称。

默认值为 ''。

.needsUpdate : boolean

当需要重新生成节点时，将此属性设为 true。

默认值为 false。

.nodeType : string

节点类型。表示该节点的结果类型（例如 float 或 vec3）。

默认值为 null。

.parents : boolean

在构建过程中为此节点创建父节点列表。

默认值为 false。

.stackTrace : string

用于调试的节点堆栈跟踪。

默认值为 null。

.type : string (readonly)

类的类型。该值通常为构造函数名称。

.updateAfterType : string

节点 Node#updateAfter 方法的更新类型。可能的取值列于 NodeUpdateType。

默认值为 'none'。

.updateBeforeType : string

节点 Node#updateBefore 方法的更新类型。可能的取值列于 NodeUpdateType。

默认值为 'none'。

.updateType : string

节点 Node#update 方法的更新类型。可能的取值列于 NodeUpdateType。

默认值为 'none'。

.uuid : string (readonly)

节点的 UUID。

.version : number (readonly)

节点的版本。当 Node#needsUpdate 设为 true 时，版本会自动递增。

默认值为 0。

.captureStackTrace : boolean

启用或禁用节点堆栈跟踪的自动捕获。

默认值为 false。

方法

.analyze( builder : NodeBuilder, output : Node )

表示分析阶段，这是构建过程的第二步，参见 Node#build 方法。 此阶段分析节点层次结构并确保后代节点被构建。

builder

当前节点构建器。
output

目标输出节点。

默认值为 null。

.build( builder : NodeBuilder, output : string | Node ) : Node | string

此方法执行节点的构建。行为和返回值取决于当前构建阶段：

  • setup：准备节点及其子节点以进行构建过程。此过程也可以创建新节点。返回节点本身或其变体。
  • analyze：分析节点层次结构以便在代码生成阶段进行优化。返回 null。
  • generate：为节点生成着色器代码。返回生成的着色器字符串。
builder

当前节点构建器。
output

可用于定义输出类型。

默认值为 null。
返回值： 构建过程的结果，取决于构建阶段。

.customCacheKey() : number

为此节点生成自定义缓存键。

返回值： 节点的缓存键。

.deserialize( json : Object )

从给定的 JSON 反序列化节点。

json

JSON 对象。

.dispose()

调用此方法会派发 dispose 事件。该事件可用于 注册用于清理任务的事件监听器。

.generate( builder : NodeBuilder, output : string ) : string

表示生成阶段，这是构建过程的第三步，参见 Node#build 方法。 此阶段构建输出节点并返回生成的着色器字符串。

builder

当前节点构建器。
output

可用于定义输出类型。
返回值： 生成的着色器字符串。

.generateNodeType( builder : NodeBuilder, output : string ) : string

返回节点的类型。

builder

当前节点构建器。
output

节点的输出。

默认值为 null。
返回值： 节点的类型。

.getArrayCount( builder : NodeBuilder ) : number

返回节点数组中的元素数量。

builder

当前节点构建器。
返回值： 节点数组中的元素数量。

.getCacheKey( force : boolean, ignores : Set.<Node> ) : number

返回此节点的缓存键。

force

设为 true 时，强制重新计算缓存键。

默认值为 false。
ignores

在计算缓存键时应忽略的节点集合。

默认值为 null。
返回值： 节点的缓存键。

.getChildren() : Node (generator)

可用于遍历子节点的生成器函数。

产出：
一个子节点。

.getElementType( builder : NodeBuilder ) : string

某些类型由多个元素组成。例如 vec3 由三个 float 值组成。此方法返回这些元素的类型。

builder

当前节点构建器。
返回值： 节点的类型。

.getHash( builder : NodeBuilder ) : string

返回用于标识节点的哈希。默认情况下它是 Node#uuid，但派生节点类可能需要根据其实现覆盖此方法。

builder

当前节点构建器。
返回值： 哈希值。

.getMemberType( builder : NodeBuilder, name : string ) : string

返回给定名称的节点成员类型。

builder

当前节点构建器。
name

成员的名称。
返回值： 节点的类型。

.getNodeType( builder : NodeBuilder, output : string ) : string

返回节点的类型。

builder

当前节点构建器。
output

节点的输出。

默认值为 null。
返回值： 节点的类型。

.getScope() : Node

返回对此节点的引用，默认为 this。

返回值： 对此节点的引用。

.getSerializeChildren() : Generator.<Object>

以 JSON 对象形式返回子节点。

返回值： 可迭代的已序列化子对象 JSON 列表。

.getShared( builder : NodeBuilder ) : Node

此方法在节点构建过程中使用，用于确保 相等的节点不会被多次构建，而只构建一次。例如，如果用户多次使用 attribute( 'uv' )，构建过程会确保只处理第一个节点。如果设置了覆盖上下文， 它还会处理节点覆盖。

builder

当前节点构建器。
返回值： 如果可能，返回共享节点；否则返回 this。

.getUpdateAfterType() : NodeUpdateType

返回 Node#updateAfter 的更新类型。

返回值： 更新类型。

.getUpdateBeforeType() : NodeUpdateType

返回 Node#updateBefore 的更新类型。

返回值： 更新类型。

.getUpdateType() : NodeUpdateType

返回 Node#update 的更新类型。

返回值： 更新类型。

.isGlobal( builder : NodeBuilder ) : boolean

默认情况下，此方法返回 Node#global 标志的值。 如果需要以分析方式确定与当前着色器阶段相关的全局缓存， 可在派生类中覆盖此方法。

builder

当前节点构建器。
返回值： 此节点是否为全局节点。

.onFrameUpdate( callback : function ) : Node

用于定义 Node#update 的便捷方法。与 Node#onUpdate 类似， 但此方法会自动将更新类型设为 FRAME。

callback

更新方法。
返回值： 对此节点的引用。

.onObjectUpdate( callback : function ) : Node

用于定义 Node#update 的便捷方法。与 Node#onUpdate 类似， 但此方法会自动将更新类型设为 OBJECT。

callback

更新方法。
返回值： 对此节点的引用。

.onReference( callback : function ) : Node

用于定义 Node#updateReference 的便捷方法。

callback

更新方法。
返回值： 对此节点的引用。

.onRenderUpdate( callback : function ) : Node

用于定义 Node#update 的便捷方法。与 Node#onUpdate 类似， 但此方法会自动将更新类型设为 RENDER。

callback

更新方法。
返回值： 对此节点的引用。

.onUpdate( callback : function, updateType : string ) : Node

用于定义 Node#update 的便捷方法。

callback

更新方法。
updateType

更新类型。
返回值： 对此节点的引用。

.serialize( json : Object )

将节点序列化为 JSON。

json

输出的 JSON 对象。

.setup( builder : NodeBuilder ) : Node

表示设置阶段，这是构建过程的第一步，参见 Node#build 方法。 派生模块通常会覆盖此方法，以准备用作节点输出/结果的节点。 如果准备了输出节点，则必须在派生模块 setup 函数的 return 语句中返回它。

builder

当前节点构建器。
返回值： 输出节点。

.toJSON( meta : Object ) : Object

将节点序列化为 three.js JSON Object/Scene 格式。

meta

可选的 JSON 对象，可能已包含其他场景对象的序列化数据。
返回值： 已序列化的节点。

.traverse( callback : traverseCallback )

可用于遍历节点的层次结构。

callback

对每个节点执行的回调。

.update( frame : NodeFrame ) : boolean (abstract)

可实现此方法，以便在节点用于渲染对象时更新其内部状态。 Node#updateType 属性定义更新的执行频率。

frame

对当前节点帧的引用。
返回值： 可选的布尔值，指示实现是否实际执行了更新（例如由于缓存而未更新）。

.updateAfter( frame : NodeFrame ) : boolean (abstract)

可实现此方法，以便在节点用于渲染对象之后更新其内部状态。 Node#updateAfterType 属性定义更新的执行频率。

frame

对当前节点帧的引用。
返回值： 可选的布尔值，指示实现是否实际执行了更新（例如由于缓存而未更新）。

.updateBefore( frame : NodeFrame ) : boolean (abstract)

可实现此方法，以便在节点用于渲染对象之前更新其内部状态。 Node#updateBeforeType 属性定义更新的执行频率。

frame

对当前节点帧的引用。
返回值： 可选的布尔值，指示实现是否实际执行了更新（例如由于缓存而未更新）。

.updateReference( state : any ) : any

节点可能引用其他对象（如材质）。此方法允许根据给定状态 （例如当前节点帧或构建器）动态更新对此类对象的引用。

state

此方法可在不同上下文中调用，因此 state 可以指代任何对象类型。
返回值： 更新后的引用。
