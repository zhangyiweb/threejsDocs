PMREMGenerator

该类从立方体贴图环境纹理生成预过滤、带多级渐远纹理的辐射环境贴图（PMREM）。这样可根据材质粗糙度快速访问不同模糊级别。结果被打包成特殊的 CubeUV 格式，以便进行自定义插值，从而支持 RGBE 等非线性格式。与传统的多级渐远链不同，它仅下降到 LOD_MIN 级别（如上所述），然后在相同的 LOD_MIN 分辨率下创建额外、过滤程度更高的“mips”，并与更高的粗糙度级别关联。这样既能保持分辨率以平滑插值漫反射光照，又能限制采样计算量。

预过滤使用基于「Sampling the GGX Distribution of Visible Normals」（Heitz, 2018）的 GGX VNDF（可见法线分布函数）重要性采样，以生成能准确匹配材质渲染中所用 GGX BRDF 的环境贴图，从而实现基于物理的基于图像的光照。

构造函数

new PMREMGenerator( renderer : WebGLRenderer )

构造一个新的 PMREM 生成器。

renderer

渲染器。

方法

.compileCubemapShader()

预编译立方体贴图着色器。可在纹理网络请求期间调用此方法以提高并发性，从而加快启动速度。

.compileEquirectangularShader()

预编译等距柱状投影着色器。可在纹理网络请求期间调用此方法以提高并发性，从而加快启动速度。

.dispose()

释放 PMREMGenerator 的内部内存。注意 PMREMGenerator 是静态类，因此通常不需要多个 PMREMGenerator 对象。若创建了多个，对其中一个调用 dispose() 也会导致其他实例不可用。

.fromCubemap( cubemap : Texture, renderTarget : WebGLRenderTarget ) : WebGLRenderTarget

从立方体贴图纹理生成 PMREM，该纹理可为 LDR 或 HDR。理想的输入立方体尺寸为 256 × 256，与 256 × 256 的立方体贴图输出最为匹配。每个面支持的最小输入立方体尺寸为 16 × 16。

cubemap

要转换的立方体贴图纹理。
renderTarget

要使用的渲染目标。

默认值为 null。
返回值： 生成的 PMREM。

.fromEquirectangular( equirectangular : Texture, renderTarget : WebGLRenderTarget ) : WebGLRenderTarget

从等距柱状投影纹理生成 PMREM，该纹理可为 LDR 或 HDR。理想的输入图像尺寸为 1k（1024 × 512），与 256 × 256 的立方体贴图输出最为匹配。支持的最小输入图像尺寸为 64 × 32。

equirectangular

要转换的等距柱状投影纹理。
renderTarget

要使用的渲染目标。

默认值为 null。
返回值： 生成的 PMREM。

.fromScene( scene : Scene, sigma : number, near : number, far : number, options : Object ) : WebGLRenderTarget

从给定场景生成 PMREM；若网络带宽较低，这可能比使用图像更快。可选的 sigma 指定在生成 PMREM 前应用于场景的模糊半径（弧度）。可选的近平面和远平面用于确保场景被完整渲染。

scene

要捕获的场景。
sigma

模糊半径（弧度）。

默认值为 0。
near

近平面距离。

默认值为 0.1。
far

远平面距离。

默认值为 100。
options

配置选项。

默认值为 {}。

size

PMREM 的纹理尺寸。

默认值为 256。
position

用于渲染场景的内部立方体相机的位置。

默认值为 origin。
返回值： 生成的 PMREM。
