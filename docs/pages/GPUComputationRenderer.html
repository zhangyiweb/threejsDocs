GPUComputationRenderer

GPUComputationRenderer，基于 @zz85 的 SimulationRenderer。

GPUComputationRenderer 使用变量（variable）的概念。这些变量是 RGBA 浮点纹理，为每个计算元素（纹素）存储 4 个浮点数。

每个变量都有一个片段着色器，用于定义获取该变量所需的计算。 你可以使用任意数量的变量，并设置依赖关系，以便在着色器中使用其他变量的纹理 （采样器 uniform 会自动添加）。大多数变量都需要将自身作为依赖项。

渲染器实际上为每个变量提供两个渲染目标，以实现乒乓（ping-pong）缓冲。当前帧的纹理 作为输入，用于渲染下一帧的纹理。

变量的渲染目标可以用作可视化着色器的输入纹理。

变量名应为合法标识符，且不应与 THREE GLSL 使用的标识符冲突。 常见做法是在变量名前加 texture 前缀，例如 texturePosition、textureVelocity...

计算的尺寸（sizeX * sizeY）会在着色器中自动定义为 resolution。例如：

#DEFINE resolution vec2( 1024.0, 1024.0 )

基本用法：

// 初始化...
// 创建计算渲染器
const gpuCompute = new GPUComputationRenderer( 1024, 1024, renderer );
// 创建初始状态浮点纹理
const pos0 = gpuCompute.createTexture();
const vel0 = gpuCompute.createTexture();
// 在此填充纹理数据...
// 添加纹理变量
const velVar = gpuCompute.addVariable( "textureVelocity", fragmentShaderVel, vel0 );
const posVar = gpuCompute.addVariable( "texturePosition", fragmentShaderPos, pos0 );
// 添加变量依赖
gpuCompute.setVariableDependencies( velVar, [ velVar, posVar ] );
gpuCompute.setVariableDependencies( posVar, [ velVar, posVar ] );
// 添加自定义 uniform
velVar.material.uniforms.time = { value: 0.0 };
// 检查完整性
const error = gpuCompute.init();
if ( error !== null ) {
		console.error( error );
}
// 在每一帧中...
// 执行计算！
gpuCompute.compute();
// 用 GPU 渲染器输出更新可视化材质中的纹理 uniform
myMaterial.uniforms.myTexture.value = gpuCompute.getCurrentRenderTarget( posVar ).texture;
// 执行渲染
renderer.render( myScene, myCamera );

此外，你可以使用工具函数创建 ShaderMaterial 并执行计算（在纹理之间渲染）。 注意着色器可以有多个输入纹理。

const myFilter1 = gpuCompute.createShaderMaterial( myFilterFragmentShader1, { theTexture: { value: null } } );
const myFilter2 = gpuCompute.createShaderMaterial( myFilterFragmentShader2, { theTexture: { value: null } } );
const inputTexture = gpuCompute.createTexture();
// 在此填充 inputTexture...
myFilter1.uniforms.theTexture.value = inputTexture;
const myRenderTarget = gpuCompute.createRenderTarget();
myFilter2.uniforms.theTexture.value = myRenderTarget.texture;
const outputRenderTarget = gpuCompute.createRenderTarget();
// 在需要的地方使用输出纹理：
myMaterial.uniforms.map.value = outputRenderTarget.texture;
// 并在每帧渲染到屏幕之前执行计算：
gpuCompute.doRenderTarget( myFilter1, myRenderTarget );
gpuCompute.doRenderTarget( myFilter2, outputRenderTarget );

导入

GPUComputationRenderer 是一个插件，必须显式导入，参见 Installation#Addons。

import { GPUComputationRenderer } from 'three/addons/misc/GPUComputationRenderer.js';

构造函数

new GPUComputationRenderer( sizeX : number, sizeY : number, renderer : WebGLRenderer )

构造一个新的 GPU 计算渲染器。

sizeX

计算问题的尺寸始终为二维：共 sizeX * sizeY 个元素。
sizeY

计算问题的尺寸始终为二维：共 sizeX * sizeY 个元素。
renderer

渲染器。

属性

.addResolutionDefine

为给定材质着色器添加 resolution 定义。

方法

.addVariable( variableName : string, computeFragmentShader : string, initialValueTexture : Texture ) : Object

向渲染器添加一个计算变量。

variableName

变量名。
computeFragmentShader

计算（片段）着色器源码。
initialValueTexture

初始值纹理。
返回值： 计算变量。

.compute()

执行计算。此方法通常在动画循环中调用。

.createRenderTarget( sizeXTexture : number, sizeYTexture : number, wrapS : number, wrapT : number, minFilter : number, magFilter : number ) : WebGLRenderTarget

根据给定参数创建新的渲染目标。

sizeXTexture

渲染目标的宽度。
sizeYTexture

渲染目标的高度。
wrapS

wrapS 值。
wrapT

wrapS 值。
minFilter

minFilter 值。
magFilter

magFilter 值。
返回值： 新的渲染目标。

.createTexture() : DataTexture

创建新的数据纹理。

返回值： 新的数据纹理。

.dispose()

释放所有内部资源。如果不再需要该渲染器，请调用此方法。

.doRenderTarget( material : Material, output : WebGLRenderTarget )

使用全屏通道将给定材质渲染到给定渲染目标。

material

材质。
output

输出。

.getAlternateRenderTarget( variable : Object ) : WebGLRenderTarget

返回给定计算变量的备用渲染目标。

variable

计算变量。
返回值： 备用渲染目标。

.getCurrentRenderTarget( variable : Object ) : WebGLRenderTarget

返回给定计算变量的当前渲染目标。

variable

计算变量。
返回值： 当前渲染目标。

.init() : string

初始化渲染器。

返回值： 若未检测到错误则返回 null，否则返回错误信息。

.renderTexture( input : Texture, output : WebGLRenderTarget )

将给定纹理渲染到给定渲染目标。

input

输入。
output

输出。

.setDataType( type : FloatType | HalfFloatType ) : GPUComputationRenderer

设置内部纹理的数据类型。

type

要设置的数据类型。
返回值： 对此渲染器的引用。

.setVariableDependencies( variable : Object, dependencies : Array.<Object> )

设置变量依赖关系。

variable

计算变量。
dependencies

表示依赖关系的其他计算变量。
