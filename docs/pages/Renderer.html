Renderer

渲染器的基类。

构造函数

new Renderer( backend : Backend, parameters : Renderer~Options )

构造一个新的渲染器。

backend

渲染器所面向的后端（例如 WebGPU 或 WebGL 2）。
parameters

配置参数。

属性

.alpha : boolean

默认帧缓冲应为透明还是不透明。

默认值为 true。

.autoClear : boolean

渲染器在执行 render() 调用前是否应自动清除当前渲染目标。目标可以是画布（默认帧缓冲）或当前绑定的渲染目标（自定义帧缓冲）。

默认值为 true。

.autoClearColor : boolean

当 autoClear 为 true 时，此属性定义渲染器是否应清除颜色缓冲。

默认值为 true。

.autoClearDepth : boolean

当 autoClear 为 true 时，此属性定义渲染器是否应清除深度缓冲。

默认值为 true。

.autoClearStencil : boolean

当 autoClear 为 true 时，此属性定义渲染器是否应清除模板缓冲。

默认值为 true。

.backend : Backend

当前后端的引用。

.contextNode : ContextNode

存储特定变换或计算覆盖节点的全局上下文节点。这些节点可用于替换渲染管线中的默认行为。

value
Object

上下文值对象。

.coordinateSystem : number (readonly)

渲染器的坐标系。此属性的值取决于所选后端，为 THREE.WebGLCoordinateSystem 或 THREE.WebGPUCoordinateSystem。

.currentColorSpace : string

渲染器的当前色彩空间。在不产生屏幕输出时，色彩空间始终为工作色彩空间。

.currentSamples : number

当前用于多重采样抗锯齿（MSAA）的采样数。

渲染到自定义渲染目标时，使用该渲染目标的采样数。若渲染器需要内部帧缓冲目标进行色调映射或色彩空间转换，则采样数设为 0。

.currentToneMapping : number

渲染器的当前色调映射。在不产生屏幕输出时，色调映射始终为 NoToneMapping。

.debug : DebugConfig

渲染器的调试配置。

.depth : boolean

默认帧缓冲是否应具有深度缓冲。

默认值为 true。

.domElement : HTMLCanvasElement | OffscreenCanvas

渲染器绘制目标画布元素的引用。此属性的值将由渲染器自动创建。

.highPrecision : boolean

启用或禁用模型视图与法线视图矩阵的高精度。启用后将使用 CPU 64 位精度以获得更高精度，而非 GPU 32 位以获得更高性能。

注意：64 位精度与 InstancedMesh 和 SkinnedMesh 不兼容。

.highPrecision : boolean

返回是否已启用高精度。

.info : Info

保存关于 GPU 内存与渲染过程的一系列统计信息。可用于调试与监控。

.initialized (readonly)

返回渲染器是否已初始化。

.inspector : InspectorBase

检查器实例。检查器可以是继承自 InspectorBase 的任意类。

.isOutputTarget

若渲染设置面向屏幕输出，则返回 true。

.isRenderer : boolean (readonly)

此标志可用于类型检测。

默认值为 true。

.library : NodeLibrary

节点库定义了材质、灯光或色调映射函数等库对象如何映射到节点类型。这是必需的，因为尽管 MeshBasicMaterial 或 PointLight 等类的实例可以是场景图的一部分，但它们在内部会表示为节点以供进一步处理。

.lighting : Lighting

用于管理灯光的类映射数据结构。

.logarithmicDepthBuffer : boolean (readonly)

是否启用对数深度缓冲。

默认值为 false。

.needsFrameBufferTarget

若需要帧缓冲目标以执行色调映射或色彩空间转换，则返回 true。若是这种情况，渲染器会为此分配内部渲染目标。

.onDeviceLost : function

定义设备/上下文丢失时应如何处理的回调函数。

.onError : function

定义报告未捕获的后端错误时应如何处理的回调函数（例如在错误作用域外抛出的 WebGPU 校验/内存不足/内部错误）。应用可重写此回调，在自身 UI 中展示错误，而不让其升级为设备丢失。默认实现会输出到控制台。

.opaque : boolean

渲染器是否应渲染不透明渲染对象。

默认值为 true。

.outputColorSpace : string

定义渲染器的输出色彩空间。

默认值为 SRGBColorSpace。

.reversedDepthBuffer : boolean (readonly)

是否启用反向深度缓冲。

默认值为 false。

.samples : number

用于多重采样抗锯齿（MSAA）的采样数。

默认值为 0。

.shadowMap : ShadowMapConfig

渲染器的阴影配置。

.sortObjects : boolean

渲染器是否应对其渲染列表排序。

注意：排序用于尝试正确渲染具有一定透明度的对象。从定义上说，对象排序并非在所有情况下都有效。根据应用需求，可能需要关闭排序并使用其他方法处理透明渲染，例如手动确定每个对象的渲染顺序。

默认值为 true。

.stencil : boolean

默认帧缓冲是否应具有模板缓冲。

默认值为 false。

.toneMapping : number

定义渲染器的色调映射。

默认值为 NoToneMapping。

.toneMappingExposure : number

定义色调映射曝光。

默认值为 1。

.transparent : boolean

渲染器是否应渲染透明渲染对象。

默认值为 true。

.xr : XRManager

渲染器的 XR 管理器。

编译给定场景中的所有材质。这有助于避免首次使用新着色器渲染对象时出现的所谓「着色器编译卡顿」（shader compilation stutter）。

若要将 3D 对象添加到已有场景，请使用第三个可选参数指定目标场景。注意：调用此方法前必须配置好（目标）场景的光照与环境。

camera

用于渲染场景的相机。
targetScene

若第一个参数是 3D 对象，则此参数必须表示该 3D 对象将要加入的场景。

默认值为 null。
返回值： 编译完成时解析的 Promise。
