Texture

所有纹理的基类。

注意：纹理首次使用后，其尺寸、格式和类型 不能更改。应改为调用 Texture#dispose 并实例化一个新纹理。

构造函数

new Texture( image : Object, mapping : number, wrapS : number, wrapT : number, magFilter : number, minFilter : number, format : number, type : number, anisotropy : number, colorSpace : string )

构造一个新的纹理。

image

保存纹理数据的图像。

默认值为 Texture.DEFAULT_IMAGE。
mapping

纹理映射方式。

默认值为 Texture.DEFAULT_MAPPING。
wrapS

wrapS 值。

默认值为 ClampToEdgeWrapping。
wrapT

wrapT 值。

默认值为 ClampToEdgeWrapping。
magFilter

放大过滤器值。

默认值为 LinearFilter。
minFilter

缩小过滤器值。

默认值为 LinearMipmapLinearFilter。
format

纹理格式。

默认值为 RGBAFormat。
type

纹理类型。

默认值为 UnsignedByteType。
anisotropy

各向异性值。

默认值为 Texture.DEFAULT_ANISOTROPY。
colorSpace

颜色空间。

默认值为 NoColorSpace。

属性

.anisotropy : number

沿穿过像素的轴、在纹素密度最高处所取的采样数。 默认情况下该值为 1。更高的值可获得比基础 mipmap 更清晰的结果，但代价是使用更多纹理采样。

默认值为 Texture.DEFAULT_ANISOTROPY。

.center : Vector2

旋转所围绕的点。值为 (0.5, 0.5) 对应 纹理中心。默认值为 (0, 0)，即左下角。

默认值为 (0,0)。

.channel : number

选择用于映射纹理的 uv 属性。0 对应 uv， 1 对应 uv1，2 对应 uv2，3 对应 uv3。

默认值为 0。

.colorSpace : string

包含颜色数据的纹理应标注为 SRGBColorSpace 或 LinearSRGBColorSpace。

默认值为 NoColorSpace。

.depth

纹理的深度（以像素为单位）。

.flipY : boolean

若设为 true，纹理在上传到 GPU 时会沿垂直轴翻转。

注意：使用 ImageBitmap 时此属性无效。需要在 创建 bitmap 时配置翻转。

默认值为 true。

.format : number

纹理的格式。

默认值为 RGBAFormat。

.generateMipmaps : boolean

是否为纹理生成 mipmap（若可能）。

若手动创建 mipmap，请将此属性设为 false。

默认值为 true。

.height

纹理的高度（以像素为单位）。

.id : number (readonly)

纹理的 ID。

.image : Object

保存纹理数据的图像对象。

.internalFormat : string

默认内部格式由 Texture#format 和 Texture#type 推导， 定义纹理数据在 GPU 上的存储方式。

此属性允许覆盖默认格式。

默认值为 null。

.isArrayTexture : boolean (readonly)

指示纹理是否应按纹理数组处理。

默认值为 false。

.isRenderTargetTexture : boolean (readonly)

指示纹理是否属于渲染目标。

默认值为 false。

.isTexture : boolean (readonly)

此标志可用于类型检测。

默认值为 true。

.magFilter : NearestFilter | NearestMipmapNearestFilter | NearestMipmapLinearFilter | LinearFilter | LinearMipmapNearestFilter | LinearMipmapLinearFilter

当一个纹素覆盖多个像素时，纹理的采样方式。

默认值为 LinearFilter。

.mapping : UVMapping | CubeReflectionMapping | CubeRefractionMapping | EquirectangularReflectionMapping | EquirectangularRefractionMapping | CubeUVReflectionMapping

纹理如何应用到对象上。默认值为 UVMapping， 即使用纹理或 uv 坐标来应用贴图。

默认值为 UVMapping。

.matrix : Matrix3

纹理的 uv 变换矩阵。

.matrixAutoUpdate : boolean

是否根据属性 Texture#offset、Texture#repeat、 Texture#rotation 和 Texture#center 更新纹理的 uv 变换 Texture#matrix。

若直接指定 uv 变换矩阵，请将此属性设为 false。

默认值为 true。

.minFilter : NearestFilter | NearestMipmapNearestFilter | NearestMipmapLinearFilter | LinearFilter | LinearMipmapNearestFilter | LinearMipmapLinearFilter

当一个纹素覆盖不到一个像素时，纹理的采样方式。

默认值为 LinearMipmapLinearFilter。

.mipmaps : Array.<Object>

保存用户自定义 mipmap 的数组。

.name : string

纹理的名称。

.needsPMREMUpdate : boolean

将此属性设为 true 表示引擎必须重新生成 PMREM。

默认值为 false。

.needsUpdate : boolean

将此属性设为 true 表示引擎必须在下一次渲染时更新纹理。 这会触发纹理上传到 GPU，并确保纹理参数配置正确。

默认值为 false。

.normalized : boolean

纹理是否应使用采样时归一化到 [0, 1] 或 [-1, 1] （取决于有符号/无符号）的 16 位整数格式之一。

默认值为 false。

.offset : Vector2

纹理单次重复在 U 和 V 方向上相对起点的偏移量。 典型范围为 0.0 到 1.0。

默认值为 (0,0)。

.onUpdate : function

回调函数，在纹理更新时调用（例如当 Texture#needsUpdate 设为 true 且随后使用该纹理时）。

默认值为 null。

.pmremVersion : number (readonly)

指示此纹理是否应由 PMREMGenerator 处理 （仅与渲染目标纹理相关）。

默认值为 0。

.premultiplyAlpha : boolean

若设为 true，在纹理上传到 GPU 时，若存在 alpha 通道， 会将其乘入颜色通道。

注意：使用 ImageBitmap 时此属性无效。需要在 创建 bitmap 时配置预乘 alpha。

默认值为 false。

.renderTarget : RenderTarget | WebGLRenderTarget

可选的反向引用，指向纹理所属的渲染目标。

默认值为 null。

.repeat : Vector2

纹理在表面 U 和 V 方向上重复的次数。若任一方向的 重复次数大于 1，对应的包裹参数也应设为 RepeatWrapping 或 MirroredRepeatWrapping 才能获得预期的平铺效果。

默认值为 (1,1)。

.rotation : number

纹理绕中心点旋转的角度（以弧度为单位）。 正值为逆时针方向。

默认值为 0。

.source : Source

纹理的数据定义。对数据源的引用可在多个纹理之间共享。 这在精灵表场景中通常很有用：多个纹理渲染相同数据， 但使用不同的纹理变换。

.type : number

纹理的数据类型。

默认值为 UnsignedByteType。

.unpackAlignment : number

指定内存中每个像素行起始处的对齐要求。 允许的值为 1（字节对齐）、2（行对齐到偶数字节）、 4（字对齐）和 8（行起始于双字边界）。

默认值为 4。

.updateRanges : Array.<Object>

可用于仅更新纹理的子区域或特定行（例如仅更新 前 3 行）。使用 addUpdateRange() 函数向此数组添加范围。

.userData : Object

可用于存储纹理自定义数据的对象。不应保存对函数的引用， 因为这些引用不会被克隆。

.uuid : string (readonly)

纹理的 UUID。

.version : number (readonly)

从 0 开始，统计 Texture#needsUpdate 被设为 true 的次数。

默认值为 0。

.width

纹理的宽度（以像素为单位）。

.wrapS : RepeatWrapping | ClampToEdgeWrapping | MirroredRepeatWrapping

定义纹理在水平方向上的包裹方式，对应 UV 映射中的 U。

默认值为 ClampToEdgeWrapping。

.wrapT : RepeatWrapping | ClampToEdgeWrapping | MirroredRepeatWrapping

定义纹理在垂直方向上的包裹方式，对应 UV 映射中的 V。

默认值为 ClampToEdgeWrapping。

.DEFAULT_ANISOTROPY : number

所有纹理的默认各向异性值。

默认值为 1。

.DEFAULT_IMAGE : Image

所有纹理的默认图像。

默认值为 null。

.DEFAULT_MAPPING : number

所有纹理的默认映射方式。

默认值为 UVMapping。

方法

.addUpdateRange( start : number, count : number )

添加要在 GPU 上更新的数据纹理数据范围。

start

开始更新的位置。
count

要更新的分量数量。

.clearUpdateRanges()

清除更新范围。

.clone() : Texture

返回一个从此实例复制数值的新纹理。

返回值： 此实例的克隆。

.copy( source : Texture ) : Texture

将给定纹理的值复制到此实例。

source

要复制的纹理。
返回值： 对此实例的引用。

.dispose()

释放此实例分配的 GPU 相关资源。当应用中 不再使用此实例时，请调用此方法。

触发：
  • Texture#event:dispose

.setValues( values : Object )

根据 values 设置此纹理的属性。

values

包含纹理参数的容器。

.toJSON( meta : Object | string ) : Object

将纹理序列化为 JSON。

meta

可选值，保存有关序列化的元信息。
参见：
  • ObjectLoader#parse
返回值： 表示序列化纹理的 JSON 对象。

.transformUv( uv : Vector2 ) : Vector2

使用纹理的 uv 变换矩阵变换给定的 uv 向量。

uv

uv 向量。
返回值： 变换后的 uv 向量。

.updateMatrix()

根据属性 Texture#offset、 Texture#repeat、Texture#rotation 和 Texture#center 更新纹理变换矩阵。

事件

.dispose

当纹理被释放时触发。

类型：
  • Object
