CubeTextureLoader

用于加载立方体纹理的类。图像在内部通过 ImageLoader 加载。

该加载器返回 CubeTexture 实例，并期望立方体贴图由 六张分别表示立方体各面的独立图像定义。不支持其他立方体贴图定义方式， 例如垂直/水平十字、列布局和行布局。

注意：按惯例，立方体贴图在指定时使用的坐标系中， 沿正 z 轴向上看时，正 x 在右侧—— 换句话说，使用的是左手坐标系。由于 three.js 使用 右手坐标系，在 three.js 中使用的环境贴图 会将正 x 与负 x 互换。

加载的立方体纹理处于 sRGB 颜色空间。即 Texture#colorSpace 默认设置为 SRGBColorSpace。

代码示例

const loader = new THREE.CubeTextureLoader().setPath( 'textures/cubeMaps/' );
const cubeTexture = await loader.loadAsync( [
	'px.png', 'nx.png', 'py.png', 'ny.png', 'pz.png', 'nz.png'
] );
scene.background = cubeTexture;

构造函数

new CubeTextureLoader( manager : LoadingManager )

构造一个新的立方体纹理加载器。

manager

加载管理器。

方法

.load( urls : Array.<string>, onLoad : function, onProgress : onProgressCallback, onError : onErrorCallback ) : CubeTexture

从指定 URL 开始加载，并将完全加载的立方体纹理 传递给 onLoad() 回调。该方法还会返回一个新的立方体纹理对象，可直接 用于创建材质。若采用这种方式，当相应加载过程完成时， 立方体纹理可能会出现在场景中。

urls

包含 6 个图像 URL 的数组，立方体纹理的每个面各一个。 URL 应按以下顺序指定：pos-x、 neg-x、pos-y、neg-y、pos-z、neg-z。也允许使用 data URI 数组。
onLoad

加载完成时执行。
onProgress

此加载器不支持。
onError

发生错误时执行。
重写： Loader#load
返回值： 立方体纹理。
