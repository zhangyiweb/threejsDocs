FileLoader

使用 Fetch API 加载资源的底层类，由大多数加载器在内部使用。 也可直接用于加载没有专用加载器的任意文件类型。

此加载器支持缓存。若要启用，请在应用程序中添加一次 THREE.Cache.enabled = true;。

代码示例

const loader = new THREE.FileLoader();
const data = await loader.loadAsync( 'example.txt' );

构造函数

new FileLoader( manager : LoadingManager )

创建一个新的文件加载器。

manager

加载管理器。

属性

.mimeType : string

期望的 MIME 类型。有效值可在 此处找到。

.responseType : 'arraybuffer' | 'blob' | 'document' | 'json' | ''

期望的响应类型。

默认值为 ''。

方法

.abort() : FileLoader

中止正在进行的 fetch 请求。

重写： Loader#abort
返回值： 此实例的引用。

.load( url : string, onLoad : function, onProgress : onProgressCallback, onError : onErrorCallback )

从给定 URL 开始加载，并将加载的响应传递给 onLoad() 回调。

url

要加载文件的路径/URL。也可以是 data URI。
onLoad

加载过程完成时执行。
onProgress

加载进行中时执行。
onError

发生错误时执行。
重写： Loader#load

.setMimeType( value : string ) : FileLoader

设置已加载文件的期望 MIME 类型。

value

MIME 类型。
返回值： 此文件加载器的引用。

.setResponseType( value : 'arraybuffer' | 'blob' | 'document' | 'json' | '' ) : FileLoader

设置期望的响应类型。

value

响应类型。
返回值： 此文件加载器的引用。
