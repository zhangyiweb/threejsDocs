BufferGeometry

用于表示网格、线段或点的几何体。它在缓冲区中包含顶点位置、面索引、法线、颜色、UV 以及自定义属性，从而降低将这些数据传递给 GPU 的开销。

代码示例

const geometry = new THREE.BufferGeometry();
// create a simple square shape. We duplicate the top left and bottom right
// vertices because each vertex needs to appear once per triangle.
const vertices = new Float32Array( [
	-1.0, -1.0,  1.0, // v0
	 1.0, -1.0,  1.0, // v1
	 1.0,  1.0,  1.0, // v2
	 1.0,  1.0,  1.0, // v3
	-1.0,  1.0,  1.0, // v4
	-1.0, -1.0,  1.0  // v5
] );
// itemSize = 3 because there are 3 values (components) per vertex
geometry.setAttribute( 'position', new THREE.BufferAttribute( vertices, 3 ) );
const material = new THREE.MeshBasicMaterial( { color: 0xff0000 } );
const mesh = new THREE.Mesh( geometry, material );

构造函数

new BufferGeometry()

构造一个新的几何体。

属性

.attributes : Object.<string, (BufferAttribute|InterleavedBufferAttribute)>

该字典以要设置的属性名称为 key，以对应的缓冲属性为 value。不要直接访问该属性，而应使用 setAttribute() 与 getAttribute() 来访问几何体的属性。

.boundingBox : Box3

几何体的包围盒，可通过 computeBoundingBox() 计算。

默认值为 null。

.boundingSphere : Sphere

几何体的包围球，可通过 computeBoundingSphere() 计算。

默认值为 null。

.drawRange : Object

决定几何体中需要渲染的部分。不应直接设置该属性，而应使用 setDrawRange()。

.groups : Array.<Object>

将几何体拆分为多个组，每个组会在一次独立的 draw call 中渲染。这样便可以在几何体上使用材质数组。

使用 addGroup() 与 clearGroups() 来编辑组，而不是直接修改该数组。

每个顶点与索引都必须且只能属于一个组——组之间不能共享顶点或索引，也不能留下未使用的顶点或索引。

.id : number (readonly)

几何体的 ID。

.index : BufferAttribute

允许在多个三角形之间复用顶点，这被称为“索引三角形”。每个三角形对应三个顶点的索引，因此该属性为每个三角面存储各顶点的索引。如果未设置该属性，渲染器会认为每三个连续的位置值表示一个三角形。

默认值为 null。

.indirect : BufferAttribute

一个（storage）缓冲属性，由计算着色器生成，并用于定义间接绘制（indirect draw calls）。

只能与 WebGPURenderer 及 WebGPU 后端一起使用。

默认值为 null。

.indirectOffset : number | Array.<number>

以字节为单位的偏移量，指向间接绘制缓冲区中数值数据开始的位置。如果提供数组，则会为每个偏移量执行一次间接绘制调用。

只能与 WebGPURenderer 及 WebGPU 后端一起使用。

默认值为 0。

.isBufferGeometry : boolean (readonly)

此标志可用于类型检测。

默认值为 true。

.morphAttributes : Object

该字典保存几何体的 morph targets。

注意：几何体一旦被渲染，morph attribute 数据就不能更改。你必须调用 dispose() 并创建新的几何体实例。

.morphTargetsRelative : boolean

用于控制 morph target 的行为；当设为 true 时，morph target 数据会被视为相对偏移，而不是绝对的位置/法线。

默认值为 false。

.name : string

几何体的名称。

.userData : Object

一个对象，可用于存储关于几何体的自定义数据。它不应持有函数引用，因为这些不会被克隆。

.uuid : string (readonly)

几何体的 UUID。

方法

.addGroup( start : number, count : number, materialIndex : number )

向该几何体添加一个组。

start

该 draw call 中的第一个元素。对于非索引几何体，它是第一个顶点；否则是第一个三角形索引。
count

指定该组包含多少个顶点（或索引）。
materialIndex

要使用的材质数组索引。

默认值为 0。

.applyMatrix4( matrix : Matrix4 ) : BufferGeometry

将指定的 4x4 变换矩阵应用到该几何体。

matrix

要应用的矩阵。
返回值： 对该实例的引用。

.applyQuaternion( q : Quaternion ) : BufferGeometry

将四元数所表示的旋转应用到该几何体。

q

要应用的四元数。
返回值： 对该实例的引用。

.center() : BufferGeometry

根据包围盒将几何体居中。

返回值： 对该实例的引用。

.clearGroups()

清除所有组。

.clone() : BufferGeometry

返回一个从该实例复制值的新几何体。

返回值： 当前实例的克隆。

.computeBoundingBox()

计算几何体的包围盒，并更新 boundingBox 成员。 引擎不会自动计算包围盒；必须由应用程序自行计算。 如果修改了几何体顶点，可能需要重新计算包围盒。

.computeBoundingSphere()

计算几何体的包围球，并更新 boundingSphere 成员。 引擎在需要时会自动计算包围球，例如用于射线检测或视锥体剔除。 如果修改了几何体顶点，可能需要重新计算包围球。

.computeTangents()

计算并为该几何体添加 tangent 属性。

仅支持已建立索引的几何体，且必须已定义 position、normal 和 uv 属性。在使用切线空间法线贴图时，请优先使用 BufferGeometryUtils#computeMikkTSpaceTangents 提供的 MikkTSpace 算法。

.computeVertexNormals()

为给定的顶点数据计算顶点法线。对于已索引的几何体，该方法将每个顶点法线设为共享该顶点的各个面的面法线平均值。 对于非索引几何体，顶点不共享，该方法将每个顶点法线设为与面法线相同。

.copy( source : BufferGeometry ) : BufferGeometry

将给定几何体的值复制到该实例。

source

要复制的几何体。
返回值： 对该实例的引用。

.deleteAttribute( name : string ) : BufferGeometry

删除指定名称的属性。

name

要删除的属性名称。
返回值： 对该实例的引用。

.dispose()

释放该实例分配的 GPU 相关资源。当应用中不再需要该实例时，应调用此方法。

触发事件：
  • BufferGeometry#event:dispose

.getAttribute( name : string ) : BufferAttribute | InterleavedBufferAttribute | undefined

返回指定名称的缓冲属性。

name

属性名称。
返回值： 缓冲属性。 若未找到属性则返回 undefined。

.getIndex() : BufferAttribute

返回该几何体的索引。

返回值： 索引。若未定义索引则返回 null。

.getIndirect() : BufferAttribute

返回该几何体的 indirect 属性。

返回值： indirect 属性。若未定义 indirect 属性则返回 null。

.hasAttribute( name : string ) : boolean

若该几何体具有指定名称的属性，则返回 true。

name

属性名称。
返回值： 该几何体是否具有指定名称的属性。

.lookAt( vector : Vector3 ) : BufferGeometry

旋转几何体以面向 3D 空间中的某个点。这通常作为一次性操作执行，而不是在循环中进行。对于典型的实时网格旋转，请使用 Object3D#lookAt。

vector

目标点。
返回值： 对该实例的引用。

.normalizeNormals()

确保几何体中的每个法线向量长度为 1。这将修正几何体表面的光照效果。

.rotateX( angle : number ) : BufferGeometry

绕 X 轴旋转几何体。这通常作为一次性操作执行，而不是在循环中进行。对于典型的实时网格旋转，请使用 Object3D#rotation。

angle

弧度制的角度。
返回值： 对该实例的引用。

.rotateY( angle : number ) : BufferGeometry

绕 Y 轴旋转几何体。这通常作为一次性操作执行，而不是在循环中进行。对于典型的实时网格旋转，请使用 Object3D#rotation。

angle

弧度制的角度。
返回值： 对该实例的引用。

.rotateZ( angle : number ) : BufferGeometry

绕 Z 轴旋转几何体。这通常作为一次性操作执行，而不是在循环中进行。对于典型的实时网格旋转，请使用 Object3D#rotation。

angle

弧度制的角度。
返回值： 对该实例的引用。

.scale( x : number, y : number, z : number ) : BufferGeometry

缩放几何体。这通常作为一次性操作执行，而不是在循环中进行。对于典型的实时网格旋转，请使用 Object3D#scale。

x

x 方向的缩放。
y

y 方向的缩放。
z

z 方向的缩放。
返回值： 对该实例的引用。

.setAttribute( name : string, attribute : BufferAttribute | InterleavedBufferAttribute ) : BufferGeometry

为指定名称设置给定属性。

name

属性名称。
attribute

要设置的属性。
返回值： 对该实例的引用。

.setDrawRange( start : number, count : number )

设置该几何体的绘制范围。

start

对于非索引几何体，是第一个顶点；否则是第一个三角形索引。
count

对于非索引 BufferGeometry，count 是要渲染的顶点数。 对于已索引 BufferGeometry，count 是要渲染的索引数。

.setFromPoints( points : Array.<Vector2> | Array.<Vector3> ) : BufferGeometry

通过基于给定点数组创建 position 属性来定义几何体。数组可包含 2D 或 3D 向量。使用二维数据时，所有顶点的 z 坐标均设为 0。

若该方法与现有的 position 属性一起使用，则会用数组中的数据覆盖顶点数据。数组长度必须与顶点数量一致。

points

点数组。
返回值： 对该实例的引用。

.setIndex( index : Array.<number> | BufferAttribute ) : BufferGeometry

为该几何体设置给定索引。

index

要设置的索引。
返回值： 对该实例的引用。

.setIndirect( indirect : BufferAttribute, indirectOffset : number | Array.<number> ) : BufferGeometry

为该几何体设置给定 indirect 属性。

indirect

保存 indirect draw calls 的属性。
indirectOffset

间接绘制缓冲区中数值数据开始位置的字节偏移量。如果提供数组，则会为每个偏移量执行一次间接绘制调用。

默认值为 0。
返回值： 对该实例的引用。

.toJSON() : Object

将该几何体序列化为 JSON。

返回值： 表示已序列化几何体的 JSON 对象。

.toNonIndexed() : BufferGeometry

返回该已索引几何体的新非索引版本。若几何体已是非索引的，则此方法不执行任何操作。

返回值： 该已索引几何体的非索引版本。

.translate( x : number, y : number, z : number ) : BufferGeometry

平移几何体。这通常作为一次性操作执行，而不是在循环中进行。对于典型的实时网格旋转，请使用 Object3D#position。

x

x 方向的偏移量。
y

y 方向的偏移量。
z

z 方向的偏移量。
返回值： 对该实例的引用。
