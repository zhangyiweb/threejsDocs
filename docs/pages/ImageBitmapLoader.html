ImageBitmapLoader

用于将图像加载为 ImageBitmap 的加载器。 ImageBitmap 提供了一种异步且资源高效的途径来准备用于渲染的纹理。

注意：使用 image bitmap 时，Texture#flipY 和 Texture#premultiplyAlpha 会被忽略。 这些选项需要通过 ImageBitmapLoader#setOptions 在加载之前配置， 与普通图像不同——普通图像可以在 Texture 上配置这些选项，以便在 GPU 上传时生效。

要匹配 Texture 的默认行为，需要以下选项：

{ imageOrientation: 'flipY', premultiplyAlpha: 'none' }

另请注意，与 FileLoader 不同，仅当启用 Cache 时，此加载器才会避免对同一 URL 发起多个并发请求。

const loader = new THREE.ImageBitmapLoader();
loader.setOptions( { imageOrientation: 'flipY' } ); // set options if needed
const imageBitmap = await loader.loadAsync( 'image.png' );
const texture = new THREE.Texture( imageBitmap );
texture.needsUpdate = true;

构造函数

new ImageBitmapLoader( manager : LoadingManager )

创建一个新的 ImageBitmap 加载器。

manager

加载管理器。

属性

.isImageBitmapLoader : boolean (readonly)

此标志可用于类型检测。

默认值为 true。

.options : Object

表示加载器选项。

默认值为 {premultiplyAlpha:'none'}。

方法

.abort() : ImageBitmapLoader

中止正在进行的 fetch 请求。

重写： Loader#abort
返回值： 此实例的引用。

.load( url : string, onLoad : function, onProgress : onProgressCallback, onError : onErrorCallback )

从给定 URL 开始加载，并将加载的 image bitmap 传递给 onLoad() 回调。

url

要加载文件的路径/URL。也可以是 data URI。
onLoad

加载过程完成时执行。
onProgress

此加载器不支持。
onError

发生错误时执行。
重写： Loader#load

.setOptions( options : Object ) : ImageBitmapLoader

设置给定的加载器选项。对象结构必须与 createImageBitmap 的 options 参数匹配。

注意：启用缓存时，缓存键仅基于 URL。使用不同选项加载同一 URL 将返回首次请求的缓存结果。

options

要设置的加载器选项。
返回值： 此 ImageBitmap 加载器的引用。
