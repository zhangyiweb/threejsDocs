CanvasTarget

CanvasTarget 是表示渲染器最终输出目标的类。

构造函数

new CanvasTarget( domElement : HTMLCanvasElement | OffscreenCanvas )

构造一个新的 CanvasTarget。

domElement

要渲染到的画布元素。

属性

.colorTexture : FramebufferTexture

默认帧缓冲的颜色纹理。

.depthTexture : DepthTexture

默认帧缓冲的深度纹理。

.domElement : HTMLCanvasElement | OffscreenCanvas

渲染器正在绘制到的画布元素的引用。 此属性的值将由渲染器自动创建。

方法

.dispose()

释放此实例分配的 GPU 相关资源。当应用中 不再使用此实例时，请调用此方法。

触发：
  • RenderTarget#event:dispose

.getDrawingBufferSize( target : Vector2 ) : Vector2

返回以物理像素为单位的绘图缓冲尺寸。此方法会考虑像素比。

target

方法将结果写入此目标对象。
返回值： 绘图缓冲尺寸。

.getPixelRatio() : number

返回像素比。

返回值： 像素比。

.getScissor( target : Vector4 ) : Vector4

返回裁剪矩形。

target

方法将结果写入此目标对象。
返回值： 裁剪矩形。

.getScissorTest() : boolean

返回裁剪测试的值。

返回值： 是否应启用裁剪测试。

.getSize( target : Vector2 ) : Vector2

返回以逻辑像素为单位的渲染器尺寸。此方法不考虑像素比。

target

方法将结果写入此目标对象。
返回值： 以逻辑像素为单位的渲染器尺寸。

.getViewport( target : Vector4 ) : Vector4

返回视口定义。

target

方法将结果写入此目标对象。
返回值： 视口定义。

.setDrawingBufferSize( width : number, height : number, pixelRatio : number )

此方法允许一次性指定宽度、高度和像素比 来定义绘图缓冲尺寸。绘图缓冲的尺寸 按以下公式计算：

size.x = width * pixelRatio;
size.y = height * pixelRatio;
width

以逻辑像素为单位的宽度。
height

以逻辑像素为单位的高度。
pixelRatio

像素比。

.setPixelRatio( value : number )

设置给定的像素比，并在必要时调整画布大小。

value

像素比。

默认值为 1。

.setScissor( x : number | Vector4, y : number, width : number, height : number )

定义裁剪矩形。

x

以逻辑像素为单位，盒子左下角的水平坐标。 除传入四个参数外，该方法也可接受单个四维向量。
y

以逻辑像素为单位，盒子左下角的垂直坐标。
width

以逻辑像素为单位的裁剪框宽度。
height

以逻辑像素为单位的裁剪框高度。

.setScissorTest( boolean : boolean )

定义裁剪测试。

boolean

是否应启用裁剪测试。

.setSize( width : number, height : number, updateStyle : boolean )

设置渲染器的尺寸。

width

以逻辑像素为单位的宽度。
height

以逻辑像素为单位的高度。
updateStyle

是否更新画布的 style 属性。

默认值为 true。

.setViewport( x : number | Vector4, y : number, width : number, height : number, minDepth : number, maxDepth : number )

定义视口。

x

以逻辑像素为单位，视口原点左下角的水平坐标。
y

以逻辑像素为单位，视口原点左下角的垂直坐标。
width

以逻辑像素为单位的视口宽度。
height

以逻辑像素为单位的视口高度。
minDepth

视口的最小深度值。仅 WebGPU。

默认值为 0。
maxDepth

视口的最大深度值。仅 WebGPU。

默认值为 1。
